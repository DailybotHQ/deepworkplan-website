# agent-create

สร้าง agent ใหม่ใน repository ปัจจุบัน เป็นตัวส่งต่อแบบบาง ๆ ไปยัง **author** sub-skill ของ Deep Work Plan

## สิ่งที่มันทำ

`agent-create` ส่งต่อไปยังขั้นตอนการสร้าง agent ของ author sub-skill agent คือนิยามของผู้ทำงานเฉพาะทาง (บทบาท ระดับโมเดล เครื่องมือ system prompt) ที่ถูกส่งไปจัดการงานประเภทหนึ่ง มันเข้ารหัสว่า "ใครทำ X" และอยู่ภายใต้ `.agents/agents/<name>.md`

## วิธีใช้

```
/agent-create
```

## พฤติกรรม

1. ตรวจหาการจัดวาง `.agents/` และข้อตกลงเฉพาะของ repo
2. ยืนยันว่ามีบทบาทที่เกิดซ้ำซึ่งต้องการโมเดล/เครื่องมือต่างออกไป แล้วตั้งชื่อแบบ kebab-case
3. สร้างโครงและกรอกข้อมูล agent (Role, Inputs, Process, Output, การ escalate)
4. เลือกระดับโมเดลที่มีเหตุผลรองรับ ทั้งแบบเบา มาตรฐาน หรือหนัก โดยไม่ hardcode รหัสโมเดลของผู้ให้บริการ
5. อัปเดตแคตตาล็อก `.agents/docs/` ให้ตรงกัน

## หมายเหตุ

สำหรับขั้นตอนการทำงานให้ใช้ `/skill-create` เก็บระดับโมเดลให้เป็นนามธรรมและ map ไว้ในคอนฟิกของ repo
