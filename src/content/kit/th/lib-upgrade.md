# lib-upgrade

อัปเกรด dependency ของ repository อย่างปลอดภัย เป็นตัวส่งต่อแบบบาง ๆ ไปยังแอดออน **dependency-upgrade**

## สิ่งที่มันทำ

`lib-upgrade` ส่งต่อไปยังแอดออน dependency-upgrade แอดออนจะให้เหตุผลเกี่ยวกับตัวจัดการแพ็กเกจจริงของ repo โดยไม่สมมติว่าเป็น npm จัดประเภทการอัปเกรดตาม semver อัปเกรดเป็นชุดที่ปลอดภัย รัน gate ตรวจสอบจริงของ repo หลังแต่ละชุด ย้อนกลับชุดที่ไม่ผ่าน แล้วสรุปผล

## วิธีใช้

```
/lib-upgrade
```

## พฤติกรรม

1. ต้องมี working tree ที่สะอาด (หรือสำรองไว้แล้ว)
2. ตรวจหาตัวจัดการแพ็กเกจจริงจาก manifest และ lockfile ที่มีอยู่
3. จัดประเภทการอัปเกรด (patch / minor / major) โดย major ต้องได้รับการอนุมัติ
4. อัปเกรดเป็นชุดย่อย และตรวจสอบ gate จริงของ repo หลังแต่ละชุด
5. ย้อนกลับชุดใดก็ตามที่ไม่ผ่าน gate แล้วสรุปผลและปล่อยให้นักพัฒนาเป็นผู้ commit

## หมายเหตุ

คำสั่งนี้ติดตั้งเฉพาะเมื่อยอมรับแอดออน dependency-upgrade ไม่ผูกกับตัวจัดการแพ็กเกจใด และไม่ commit ให้โดยอัตโนมัติ
