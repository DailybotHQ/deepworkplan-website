# พรีเซ็ต Generic

แนวทางการให้เหตุผลสำรองที่ขั้นตอน onboard ใช้เมื่อไม่มีพรีเซ็ตเฉพาะสแตกใดที่ตรงกัน นี่คือเช็กลิสต์ ไม่ใช่เทมเพลต โดยยึดสภาพจริงที่ตรวจพบเป็นหลัก

## สิ่งที่ต้องพิจารณา

- ตรวจหาคำสั่ง build และ test จริงจาก manifest, Makefile หรือ CI แล้วบันทึกไว้ตามต้นฉบับทุกตัวอักษร
- ใช้แนวปฏิบัติทั่วไป โดยไม่ตั้งสมมติฐานที่ผูกกับสแตกใดเป็นการเฉพาะ
- สร้างเฉพาะ skill, agent และเอกสารที่สอดคล้องกับเวิร์กโฟลว์จริงของ repo เท่านั้น

## หมายเหตุ

ตัวสำรองที่ใช้เมื่อไม่ตรงกับ Django, Vue + Vite, Astro/Svelte, Node/TS service หรือ Python package/CLI เลย
