# adapter Cloud agents

Cloud และ background agent — Claude Code remote tasks, OpenAI Codex cloud, agent ประเภท Jules และสภาพแวดล้อมการดำเนินงานแบบ asynchronous ที่คล้ายกัน — เป็นแบบชั่วคราวโดยการออกแบบ: เซสชันเริ่มต้น รันในเวลาจำกัด และสิ้นสุด DWP เปลี่ยนข้อจำกัดนั้นให้เป็นคุณสมบัติโดยทำให้แผนเป็นสิ่งประดิษฐ์ที่ทนทาน ไม่ใช่เซสชัน

## ข้อคิดเห็นหลัก

ไดเรกทอรี `.dwp/` ของ repository คือ**ชั้น spec และสถานะที่ทนทาน** Cloud session คือ**ตัวดำเนินการชั่วคราว** แผนบอกตัวดำเนินการว่าต้องทำอะไร `state.json` บอกว่าต้อง resume ที่ใด เซสชันสามารถเป็นอะไรก็ได้ — โมเดลอื่น provider อื่น container ใหม่ — และแผนดำเนินต่อจากจุดที่หยุดไว้ได้อย่างแม่นยำ

## ระดับการสนับสนุน

**Full** — cloud agent ใดก็ตามที่อ่าน repository ปฏิบัติตาม `AGENTS.md` และสามารถดำเนินคำสั่ง shell รองรับ DWP โดยไม่ต้องมี adapter เฉพาะแพลตฟอร์ม

## ลูป wake-resume-yield

แต่ละรอบ cloud agent ปฏิบัติตามรูปแบบ:

1. **Wake** — เซสชันเริ่มต้น (ทริกเกอร์โดยกำหนดเวลา cron เหตุการณ์ CI หรือการกระทำของผู้ใช้)
2. **รัน DWP Resume Protocol** — อ่าน README ของแผน ระบุ checkpoint ปรับประสาน `state.json` กับ markdown ตรวจสอบรอยต่อ รัน smoke test ดู [ข้อกำหนด DWP](/spec/dwp-specification#the-dwp-resume-protocol)
3. **ตรวจสอบสถานะ blocked** — ถ้า `state.json.blocked` ถูกตั้งค่าไว้ รายงานเหตุผลและ yield โดยไม่ดำเนินการ
4. **ดำเนินงาน atomic ถัดไป** — หนึ่งงาน ผ่าน validation gate เต็มรูปแบบ
5. **อัปเดตชั้นสถานะ** — เขียน `state.json` แบบ atomic ใหม่พร้อมสถานะงานใหม่ บันทึก gate บันทึกผลลัพธ์ และ commit hash
6. **Yield** — เซสชันสิ้นสุดอย่างสะอาด รอบถัดไปรับต่อจากสถานะที่อัปเดต

## ข้อกำหนด

- แผน MUST พกชั้นสถานะที่เครื่องอ่านได้ (`manifest.json` + `state.json`) หากไม่มี เซสชันชั่วคราวไม่มีทางที่น่าเชื่อถือในการกำหนดตำแหน่ง resume ดู [สถานะแผน](/spec/plan-state)
- แผน MUST ได้รับการอนุมัติจากมนุษย์ก่อนรอบ cloud ที่ไม่มีผู้ดูแลใด ๆ การอนุมัติแผนคือจุดควบคุม
- เอเจนต์ทำงานภายใต้อำนาจจำกัด: มัน MUST NOT ขยายขอบเขต ดำเนินการทำลายหรือส่งออกภายนอกที่แผนไม่ได้อนุญาตอย่างชัดเจน หรือด้นสดเกินกว่าคำสั่งของงานปัจจุบัน ดู [โปรโตคอลของเอเจนต์](/spec/agent-protocol#execution-profiles)

## หมายเหตุ

Cloud agent session คือกรณีสุดโต่งของโปรไฟล์การดำเนินงานแบบไม่มีผู้ดูแล: ไม่มีความต่อเนื่องของเซสชันเลย ความทนทานของแผนสูงสุด ไดเรกทอรี `.dwp/` ของ DWP ได้รับการออกแบบมาเพื่อให้ทำงานได้โดยเฉพาะ — สถานะทั้งหมดที่เอเจนต์ต้องการเพื่อเริ่ม ดำเนินต่อ และเสร็จสิ้นงานอยู่ใน markdown ธรรมดาและไฟล์ JSON ที่เซสชันใดก็ตามสามารถอ่านได้จากการ checkout ใหม่
