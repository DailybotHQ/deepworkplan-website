# แอดออน Dependency-upgrade

นำ dependency ของ repository ให้ทันสมัยด้วยเวิร์กโฟลว์แบบทีละชุด ตรวจสอบได้ และย้อนกลับได้ นี่คือแอดออนแบบสมัครใจตัวที่สามของ Deep Work Plan

## สิ่งที่เพิ่มเข้ามา

- การตรวจหาตัวจัดการแพ็กเกจ **จริง** ของ repo (npm/pnpm/yarn + ncu, pip/poetry/uv, cargo, go mod, bundler, composer และอื่น ๆ) จาก manifest และ lockfile ที่มีอยู่
- คำสั่ง `/lib-upgrade` ที่ติดตั้งลงใน `.agents/commands/` **เฉพาะเมื่อยอมรับเท่านั้น**

## พฤติกรรม

1. ต้องมี working tree ที่สะอาด (หรือสำรองไว้แล้ว)
2. จัดประเภทการอัปเกรดตาม semver ทั้ง patch, minor, major โดย major ต้องได้รับการอนุมัติอย่างชัดเจน
3. อัปเกรดเป็นชุดย่อยที่เกี่ยวเนื่องกัน ไม่ทำทั้งหมดในคราวเดียว
4. รัน gate ตรวจสอบจริงของ repo หลังแต่ละชุด และย้อนกลับชุดใดก็ตามที่ไม่ผ่าน
5. สรุปว่ามีอะไรอัปเกรด ข้าม และย้อนกลับบ้าง แล้วปล่อยให้นักพัฒนาเป็นผู้ commit

## หมายเหตุ

ไม่ผูกกับตัวจัดการแพ็กเกจใด จะไม่รัน npm/ncu ใน repo ที่ไม่ใช่ JS เป็นแบบสมัครใจ ไม่บังคับเสมอ และไม่ commit ให้โดยอัตโนมัติ
