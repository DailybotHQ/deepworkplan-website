# adapter Hermes

Hermes คือแพลตฟอร์มเอเจนต์อัตโนมัติที่โหลด skill ผ่านมาตรฐาน AgentSkills และรักษาหน่วยความจำเชิงลำดับเหตุการณ์ข้ามเซสชัน DWP ผสานรวมกับทั้งสองพื้นผิว: skill pack โหลดจาก `.agents/skills/` ในลักษณะเดียวกับ pack ที่เข้ากันได้กับ AgentSkills อื่น ๆ และบันทึกผลลัพธ์ของ `state.json` ป้อนข้อมูลโดยตรงเข้าสู่ดัชนีหน่วยความจำของ Hermes

## ระดับการสนับสนุน

**Full** — Hermes อ่าน `AGENTS.md` และโหลด skill ผ่านการโหลดตามมาตรฐาน AgentSkills sub-skill DWP ทั้งหมดพร้อมใช้งาน

## การติดตั้ง

ติดตั้ง skill pack ลงในไดเรกทอรี `.agents/skills/` ของ workspace หรือใช้ตัวจัดการ skill ของแพลตฟอร์มหากมี

## การเรียกใช้

ข้อความธรรมดา:

```
create a plan for <goal>
resume the open plan
```

## ความสอดประสานของหน่วยความจำเชิงลำดับเหตุการณ์

Hermes จัดทำดัชนีหน่วยความจำของเอเจนต์เพื่อให้งานที่ผ่านมาค้นคืนได้ข้ามเซสชันในอนาคต บันทึกผลลัพธ์ `state.json` ของ DWP ได้รับการออกแบบมาสำหรับการใช้งานนี้โดยเฉพาะ: งานที่เสร็จแล้วแต่ละงานพกสิ่งที่ `tried` สิ่งที่ `failed` สิ่งที่ `worked` และหมายเหตุรูปแบบอิสระในฟิลด์ JSON ที่มีโครงสร้าง เมื่อ Hermes จัดทำดัชนีไฟล์สถานะ `.dwp/plans/` ของ workspace แผนที่เสร็จแล้วจะกลายเป็นหน่วยความจำเชิงลำดับเหตุการณ์ที่ค้นคืนได้ — เอเจนต์สามารถค้นคืนได้ในภายหลังว่าปัญหาเฉพาะถูกแก้ไขอย่างไร ไม่ใช่แค่ว่าแผนทำงาน

ซึ่งหมายความว่าแผน DWP บน Hermes มีความทนทานสองทาง: ไฟล์ markdown คือบันทึกที่มนุษย์อ่านได้ และบันทึกผลลัพธ์ `state.json` คือหน่วยความจำที่จัดทำดัชนีได้ด้วยเครื่องจักร ดู [สถานะแผน](/spec/plan-state#outcome-records-as-episodic-memory)

## การดำเนินต่อด้วย cron

บนแพลตฟอร์มที่มีการกำหนดเวลา cron การดำเนินต่อเป็นไปตามรูปแบบเดียวกับแพลตฟอร์มอัตโนมัติอื่น ๆ: แต่ละรอบที่กำหนดเวลาตื่น รัน DWP Resume Protocol ดำเนินอย่างมากหนึ่งงาน อัปเดต `state.json` และ yield แผน ไม่ใช่เซสชัน คือหน่วยของความต่อเนื่อง

ชั้นสถานะที่เครื่องอ่านได้ (`manifest.json` + `state.json`) REQUIRED สำหรับการดำเนินงานแบบไม่มีผู้ดูแล ดู [โปรโตคอลของเอเจนต์](/spec/agent-protocol#execution-profiles) สำหรับโปรโตคอลเงื่อนไขหยุดและการส่งต่อเพื่อแก้ไขฉบับเต็ม

## หมายเหตุ

Hermes และ OpenClaw ต่างเป็นแพลตฟอร์มเอเจนต์อัตโนมัติในตารางเอเจนต์ที่รองรับใน [โปรโตคอลของเอเจนต์](/spec/agent-protocol) ทั้งคู่รันภายใต้โปรไฟล์การดำเนินงานแบบไม่มีผู้ดูแล และทั้งคู่ได้ประโยชน์จากความเทียบเท่า `state.json`-as-git-log ของ archetype agent workspace
