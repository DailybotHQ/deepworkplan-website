# lib-upgrade

रिपॉज़िटरी की dependencies को सुरक्षित रूप से अपग्रेड करें। यह **dependency-upgrade** ऐडऑन के लिए एक पतला डेलिगेटर है।

## यह क्या करता है

`lib-upgrade` dependency-upgrade ऐडऑन तक रूट करता है। ऐडऑन रिपॉज़िटरी के वास्तविक package manager के बारे में तर्क करता है — कभी npm मानता नहीं — अपग्रेड को semver द्वारा वर्गीकृत करता है, सुरक्षित बैचों में अपग्रेड करता है, हर बैच के बाद रिपॉज़िटरी का वास्तविक validation gate चलाता है, विफल बैच को वापस लौटाता है, और सारांश देता है।

## उपयोग

```
/lib-upgrade
```

## व्यवहार

1. एक स्वच्छ (या बैकअप किया हुआ) working tree आवश्यक है।
2. मौजूद manifest और lockfile से वास्तविक package manager का पता लगाएं।
3. अपग्रेड वर्गीकृत करें (patch / minor / major); major के लिए अनुमोदन आवश्यक है।
4. छोटे बैचों में अपग्रेड करें; हर एक के बाद रिपॉज़िटरी के वास्तविक gate को मान्य करें।
5. gate में विफल होने वाले किसी भी बैच को वापस लौटाएं; सारांश दें और डेवलपर को कमिट करने दें।

## टिप्पणियाँ

यह कमांड केवल तभी इंस्टॉल होता है जब dependency-upgrade ऐडऑन स्वीकृत हो। Package-manager agnostic; यह कभी स्वतः कमिट नहीं करता।
