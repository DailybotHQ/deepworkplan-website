# Cloud agents adapter

क्लाउड और पृष्ठभूमि एजेंट्स — Claude Code remote tasks, OpenAI Codex cloud, Jules-class एजेंट्स, और इसी तरह के असंयोज़ित निष्पादन परिवेश — डिज़ाइन से अल्पकालिक हैं: एक सत्र शुरू होता है, सीमित समय के लिए चलता है, और समाप्त होता है। DWP उस बाधा को एक सुविधा में बदल देता है: योजना को टिकाऊ आर्टिफ़ैक्ट बनाकर, सत्र को नहीं।

## मूल अंतर्दृष्टि

रिपॉज़िटरी की `.dwp/` डायरेक्टरी **टिकाऊ spec और state परत** है। क्लाउड सत्र **अल्पकालिक निष्पादक** हैं। योजना निष्पादक को बताती है क्या करना है; `state.json` उसे बताता है कहाँ से फिर शुरू करना है। सत्र कुछ भी हो सकता है — एक अलग मॉडल, एक अलग provider, एक नया container — और योजना ठीक वहीं से जारी रहती है जहाँ छूटी थी।

## समर्थन स्तर

**पूर्ण** — कोई भी क्लाउड एजेंट जो रिपॉज़िटरी पढ़ता है, `AGENTS.md` का पालन करता है, और shell commands निष्पादित कर सकता है, वह प्लेटफ़ॉर्म-विशिष्ट adapter के बिना DWP का समर्थन करता है।

## wake-resume-yield लूप

प्रत्येक क्लाउड एजेंट turn इस पैटर्न का पालन करता है:

1. **Wake** — सत्र शुरू होता है (cron शेड्यूल, CI इवेंट, या उपयोगकर्ता क्रिया द्वारा ट्रिगर)।
2. **DWP Resume Protocol चलाएँ** — योजना README पढ़ें, चेकपॉइंट ढूँढें, markdown के विरुद्ध `state.json` का पुनर्मेल करें, seam का निरीक्षण करें, smoke test चलाएँ। देखें [DWP विनिर्देश](/spec/dwp-specification#the-dwp-resume-protocol)।
3. **अवरुद्ध स्थिति जाँचें** — यदि `state.json.blocked` सेट है, तो कारण रिपोर्ट करें और निष्पादन के बिना yield करें।
4. **अगला परमाणु कार्य निष्पादित करें** — एक कार्य, अपना पूर्ण सत्यापन गेट पार करते हुए।
5. **स्थिति परत अद्यतन करें** — नई कार्य स्थिति, गेट रिकॉर्ड, परिणाम रिकॉर्ड, और commit hash के साथ `state.json` परमाणु रूप से पुनर्लिखें।
6. **Yield** — सत्र cleanly समाप्त होता है; अगला turn अद्यतन स्थिति से उठाता है।

## आवश्यकताएँ

- योजना में मशीन-पठनीय स्थिति परत (`manifest.json` + `state.json`) MUST होनी चाहिए। इसके बिना, एक अल्पकालिक सत्र के पास रिज्यूम स्थिति निर्धारित करने का कोई विश्वसनीय तरीका नहीं है। देखें [योजना-स्थिति](/spec/plan-state)।
- किसी भी अनुपस्थित क्लाउड turn से पहले MUST एक मानव द्वारा योजना अनुमोदित की जानी चाहिए। योजना अनुमोदन नियंत्रण बिंदु है।
- एजेंट सीमित अधिकार के अंतर्गत संचालित होता है: इसे MUST NOT दायरा विस्तारित करना चाहिए, ऐसे विनाशकारी कार्य करने चाहिए जिन्हें योजना स्पष्ट रूप से अधिकृत नहीं करती, या वर्तमान कार्य के निर्देशों से परे सुधार करना चाहिए। देखें [एजेंट प्रोटोकॉल](/spec/agent-protocol#execution-profiles)।

## टिप्पणियाँ

क्लाउड एजेंट सत्र अनुपस्थित निष्पादन प्रोफ़ाइल का चरम मामला हैं: शून्य सत्र निरंतरता, अधिकतम योजना स्थायित्व। DWP की `.dwp/` डायरेक्टरी विशेष रूप से इसे काम करने के लिए डिज़ाइन की गई है — सभी state जिसकी एक एजेंट को काम का एक निकाय शुरू करने, जारी रखने और पूर्ण करने के लिए आवश्यकता है, वह plain markdown और JSON फ़ाइलों में रहता है जिन्हें कोई भी सत्र एक नई checkout से पढ़ सकता है।
