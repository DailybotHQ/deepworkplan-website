# Дополнение AI Diff Reviewer

Подключает выполнение Deep Work Plan к **[AI Diff Reviewer](https://github.com/DailybotHQ/ai-diff-reviewer)** (публикация в маркетплейсе — **«AI Diff Reviewer»**, текущая версия **v2.0.0**), чтобы обязательная финальная задача **Проверки безопасности** получила структурированную локальную проверку — вердикт, таблицу находок и уровень серьёзности — а при выборе Flow B каждый pull request мог закрываться той же проверкой в CI. Дополнение по **выбору**; никогда не требуется для соответствия AI-first.

Основная методология Deep Work Plan имеет **нулевую** зависимость от AI Diff Reviewer. Репозиторий без дополнений полностью соответствует требованиям. Предлагайте это дополнение только тогда, когда разработчик или команда хочет структурированного качества проверки; никогда не устанавливайте автоматически для всех. Всегда уточняйте Flow A или Flow B — никогда не используйте значение по умолчанию.

## Когда использовать

| Сигнал | Действие |
|--------|--------|
| Команда хочет CI-шлюз для слияния PR со структурированными находками | **Рекомендовать Flow B** при onboarding |
| Личный или экспериментальный репозиторий; локальная проверка перед push достаточна | **Предложить Flow A** |
| Нет потребности в дополнительной поверхности проверки | **Пропустить** — базовая Проверка безопасности по-прежнему действует |

## Два потока принятия

| Поток | Что вы получаете |
|------|----------------|
| **A — только локально** | Встроенный skill + обязательный `.review/extension.md` (через `generate-extension`). Расширяет Проверку безопасности локальным проходом. Без рабочего процесса GitHub Actions. |
| **B — двойная поверхность** | Flow A плюс `setup` записывает `.github/workflows/pr-review.yml` (Action `@v2`), тот же файл расширения для локальной и CI-проверки. Опциональный компаньон `apply-review` после того, как CI публикует находки. |

Для обнаружения расширения Проверки безопасности требуется **skill + файл расширения** в одном из мест: `.review/extension.md`, `.github/ai-diff-reviewer/extension.md` или `.github/ai-pr-reviewer/extension.md`. Одного skill недостаточно.

## Что подключает это дополнение (намеренно ограничено)

Дополнение DWP **не** изобретает проверяльщика заново. Оно делегирует установку, методологию, мастер CI, создание расширений, составление PR и обход результатов после CI пяти подскиллам вышестоящего skill (родительский поток по умолчанию, `generate-extension`, `setup`, `open-pr`, `apply-review`).

### Расширение Проверки безопасности

При обнаружении `create` / `execute` добавляют шаг локальной проверки к обязательной задаче Проверки безопасности. Вывод добавляется под `## AI Diff Reviewer local review` в `analysis_results/SECURITY_REVIEW.md`.

- **Мягкий сбой (только вызов):** отсутствующий skill, отсутствующее расширение или ошибка вызова → предупредить один раз и продолжить; никогда не проваливать задачу из-за этого.
- **Шлюз после завершённого прохода:** находки `critical` по-прежнему блокируют завершение Проверки безопасности до исправления или явного принятия (существующий SR-контракт). `warning` / `info` документируются, но не блокируют.
- **Flow A не требует CI-секрета.** Незаданный `CURSOR_API_KEY` не должен подавлять локальный проход.

### CI-шлюз Flow B (опционально)

Закреплённый Action `DailybotHQ/ai-diff-reviewer@v2`, обычно блокированный меткой (`ready`), со стабильно именованным заданием **AI review gate** для защиты ветки и опциональной меткой пропуска `skip-review-label: skip-ai-review`. Общие `prompt.md` + расширение согласовывают методологию и уровень серьёзности; при итерационно-осознанной проверке раунды CI 2+ могут быть короче, тогда как локальный проход остаётся полным.

### Опциональный компаньон `apply-review`

После того как CI публикует проверку, разработчик может вызвать `apply-review` во время `execute`, чтобы пройти по находкам по одной (применить / отложить / пропустить) с согласия. Только чтение по умолчанию; никогда не является файлом задачи плана (нарушит обязательный порядок финальных задач).

## Поведение

- **Уточнять поток; никогда не угадывать.** Установка рабочего процесса без запроса создаёт большую нагрузку, чем остаться на Flow A.
- **Согласовывать, не затирать.** Существующий skill, расширение или `pr-review.yml` сохраняются; заполнять только пробелы.
- **Аутентификация отложена.** Секреты провайдера для CI настраивает сопровождающий; это дополнение никогда не хранит учётные данные.
- **Нейтральность к провайдеру.** Отказ оставляет полностью AI-first репозиторий.

## Примечания

По выбору и никогда не обязательно. Вышестоящий skill: [DailybotHQ/ai-diff-reviewer](https://github.com/DailybotHQ/ai-diff-reviewer). Страница спецификации: [Add-ons](/spec/addons).
