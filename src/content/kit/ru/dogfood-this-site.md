# Разбор случая: этот сайт

Сайт, который вы читаете, работает на методологии, которую он документирует. Он dogfood-ит Deep Work Plan: тот же навык, тот же процесс `/init`, те же планы `.dwp/`, которые использовал бы любой другой репозиторий. Это реальный рассказ, а не гипотеза.

## До

Как и большинство репозиториев, этот не был построен для агентов. Контекст жил в головах людей и разрозненных заметках, не было единого источника истины, который агент мог бы прочитать, а онбординг нового агента означал повторное объяснение проекта каждую сессию. Долгосрочная работа отклонялась от курса.

## Принятие DWP

Репозиторий был сделан AI-first одним Deep Work Plan, разложенным на атомарные задачи с validation gates:

1. Установить навык Deep Work Plan как эталонную установку, закреплённую `skills-lock.json`.
2. Запустить онбординг, чтобы создать осмысленный `AGENTS.md`, дерево `docs/` и документацию по модулям из реального стека репозитория.
3. Построить межагентный набор `.agents/` — тонкие командные делегаторы `dwp-*` и каталог, соответствующий тому, что есть на диске.
4. Развернуть игнорируемое git-ом рабочее пространство `.dwp/` для планов и черновиков.
5. Проверить соответствие с помощью `/dwp-verify`.

Каждая задача проверялась относительно реальных gates репозитория — `biome`, `astro:check`, набор тестов, продакшен-сборка и проверка паритета эндпоинтов для агентов — прежде чем была отмечена завершённой.

## После

Репозиторий теперь AI-first по его собственной спецификации: осмысленный `AGENTS.md`, упорядоченное по категориям дерево `docs/`, межагентный набор `.agents/` и игнорируемое git-ом рабочее пространство `.dwp/`. Любой агент — Claude Code, Cursor, Codex, Gemini, Copilot — может открыть его, прочитать harness и выполнять долгосрочные планы без сопровождения за руку каждую сессию.

## Результат

Методология доказывает себя на собственном исходном коде: этот сайт строится и сопровождается тем же способом, каким он советует строить ваш, — следуя [`/init.md`](/init.md). Если стандарт работает здесь, в продакшене, он сработает и для вашего репозитория.
