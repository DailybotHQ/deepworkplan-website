# skill-create

Создаёт новый навык в текущем репозитории. Тонкий делегатор к под-навыку **author** Deep Work Plan.

## Что он делает

`skill-create` маршрутизирует к процессу создания навыка под-навыка author. Навык — это переиспользуемая, параметризованная процедура, выполняемая внутри сессии (например, `/fix-lint`). Он кодирует «как делать X» и живёт в `.agents/skills/<name>/SKILL.md`.

## Использование

```
/skill-create
```

## Поведение

1. Определить структуру `.agents/` репозитория и локальные соглашения.
2. Подтвердить, что существует реальный, повторяемый рабочий процесс; назвать его в kebab-case.
3. Развернуть и заполнить `SKILL.md` (Goal, When-to-use, Steps, Validation).
4. При необходимости добавить тонкий командный делегатор.
5. Обновить каталог `.agents/docs/`, чтобы он соответствовал.

## Примечания

Для агентов используйте `/agent-create`. Логика живёт в навыке; команда остаётся тонкой.
