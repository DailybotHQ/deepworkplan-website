# Адаптер OpenClaw

OpenClaw — автономная агентная платформа с долгоживущими рабочими пространствами и запланированными поворотами. Она нативно сканирует `<workspace>/.agents/skills/` для установленных пакетов навыков, что означает: пакет навыков DWP загружается без какой-либо адаптерной прослойки — установите его туда, и OpenClaw подхватит его автоматически.

## Уровень поддержки

**Полная** — OpenClaw читает `AGENTS.md` и загружает навык через стандарт AgentSkills. Все под-навыки DWP (`onboard`, `create`, `execute`, `refine`, `resume`, `status`, `verify`) доступны.

## Установка

```bash
openclaw skills install deepworkplan
```

Или вручную: клонировать `DailybotHQ/deepworkplan-skill` в `<workspace>/.agents/skills/deepworkplan/`.

## Вызов

Простой текст — OpenClaw не использует префикс слеш-команды:

```
onboard this workspace with deepworkplan
create a plan for <goal>
resume the open plan
```

## Маппинг автономного выполнения

Примитивы планирования OpenClaw напрямую соответствуют модели продолжения DWP:

| Примитив OpenClaw | Роль в DWP |
|-------------------|-----------|
| Корень рабочего пространства | Архетип [рабочего пространства агента](/spec/archetypes): `AGENTS.md`, `.agents/`, `.dwp/` в корне рабочего пространства |
| `<workspace>/.agents/skills/` (нативное сканирование tier-2) | Где живёт этот пакет — адаптерная прослойка не нужна |
| Heartbeat или cron-поворот | Один поворот [запланированного продолжения](/spec/agent-protocol#scheduled-continuation): пробуждение → Протокол возобновления DWP → следующая атомарная задача → обновление состояния → yield |
| Проверки `HEARTBEAT.md` | Добавьте одну строку: «Если в `.dwp/plans/` есть открытый план, возобновить его на одну задачу.» |
| Постоянные инструкции | Граница одобрения плана и ограниченные полномочия автономного профиля |

Операционно:

1. Человек интерактивно создаёт и утверждает план. Одобрение — это контрольная точка: автономные повороты никогда не создают и не выполняют план в одном повороте.
2. План MUST нести машиночитаемый слой состояния (`manifest.json` + `state.json`). В рабочем пространстве без git именно `state.json` делает возможным безопасное возобновление после сбоя. См. [Состояние плана](/spec/plan-state).
3. Каждый heartbeat или cron-поворот выполняет не более одной задачи, проходит её validation gate, атомарно обновляет `state.json` и завершается.
4. При любом условии остановки агент записывает поле `blocked` в `state.json` и сообщает через штатный канал рабочего пространства. Следующий взгляд человека — или heartbeat-отчёт — сразу видит, что нужно.

В результате: многодневный план, переживающий перезапуски, смену модели и границы сессий, выполняемый демоном за ночь — с теми же gates, что и при выполнении под наблюдением человека.

## Примечания

Рабочее пространство OpenClaw — канонический пример архетипа [рабочего пространства агента](/spec/archetypes). Интеграция с `HEARTBEAT.md` и требование `state.json` в пакете навыков DWP разработаны с учётом этой платформы.
