# dwp-create

Создание нового Deep Work Plan из высокоуровневой цели.

<CmdCreate lang="ru" class="not-prose my-8" />

## Что он делает

`dwp-create` берёт высокоуровневую цель и превращает её в структурированный план. Он анализирует цель, при необходимости задаёт уточняющие вопросы, раскладывает работу на последовательные задачи и пишет файлы плана.

## Использование

```
/dwp-create <goal>
```

## Режимы

`dwp-create` по умолчанию работает в **управляемом** режиме: он собирает контекст, пишет единый доработанный черновик, **показывает его вам и запрашивает подтверждение перед созданием финального плана** — так что вы всегда читаете план до того, как он создан. Это рекомендуемый режим по умолчанию, особенно в первые несколько раз.

Добавьте **`trust`** (или `auto`) последним словом, чтобы пропустить подтверждения — агент пишет доработанный черновик и переходит прямо к финальному плану:

```
/dwp-create <goal> trust
```

Режим trust — это сокращение для опытных пользователей, когда вы доверяете процессу; управляемый режим по умолчанию оставляет человека в цикле.

## Поведение

1. Проанализировать цель и контекст репозитория.
2. Задать уточняющие вопросы, если есть неоднозначность.
3. Написать доработанный черновик в `.dwp/drafts/` (в управляемом режиме проверить и подтвердить его).
4. Разложить на последовательные задачи и написать README, файлы задач и PROGRESS.
5. Завершить тремя обязательными финальными задачами (Security Review, Skills & Agents Discovery, Executive Report).

## Псевдонимы

- `dwp:create`
