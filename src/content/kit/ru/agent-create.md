# agent-create

Создаёт нового агента в текущем репозитории. Тонкий делегатор к под-навыку **author** Deep Work Plan.

## Что он делает

`agent-create` маршрутизирует к процессу создания агента под-навыка author. Агент — это определение специализированного исполнителя (роль, уровень модели, инструменты, системный промпт), направляемого для обработки класса задач. Он кодирует «кто делает X» и живёт в `.agents/agents/<name>.md`.

## Использование

```
/agent-create
```

## Поведение

1. Определить структуру `.agents/` репозитория и локальные соглашения.
2. Подтвердить повторяющуюся роль с отдельными потребностями в модели/инструментах; назвать её в kebab-case.
3. Развернуть и заполнить агента (Role, Inputs, Process, Output, эскалация).
4. Выбрать обоснованный уровень модели — лёгкий, стандартный или тяжёлый — без жёсткого прописывания ID моделей вендоров.
5. Обновить каталог `.agents/docs/`, чтобы он соответствовал.

## Примечания

Для процедур используйте `/skill-create`. Держите уровни моделей абстрактными и сопоставляйте их в конфигурации репозитория.
