# 디자인 시스템 애드온

프런트엔드/UI 리포지토리에 `DESIGN.md`를 부여합니다 — 어떤 코딩 에이전트든 읽어서, 안내가 없을 때 에이전트가 기대는 스타일 없는 통계적 기본값 대신 리포지토리 **고유의** 디자인 시스템에 일관된 UI를 생성하게 하는 Markdown 디자인 시스템 파일입니다. 네 번째 선택형 Deep Work Plan 애드온입니다.

## 무엇을 추가하는가

- `docs/DESIGN.md`에 위치하는 `DESIGN.md`(리포지토리의 다른 스펙들과 나란히 두며, `docs/` 트리가 없을 때만 리포지토리 루트에 둠). 에이전트가 나머지 문서처럼 발견할 수 있도록 **`AGENTS.md`에서 참조**됩니다.
- 정규 섹션들: 개요/분위기, 색상 팔레트 & 역할(라이트 + 다크), 타이포그래피, 레이아웃 & 간격, 입체감 & 깊이, 형태, 컴포넌트, 반응형 동작, 권장 사항 & 금지 사항(리포지토리의 접근성 규칙 포함), 그리고 에이전트 프롬프트 가이드.
- 파일이 완전한지, 문서화된 텍스트 대비가 WCAG AA를 충족하는지, 토큰 참조가 해석되는지 확인하는 검증 단계.

## 동작

- **추론하라, 복사하지 마라.** 모든 값은 리포지토리의 실제 디자인 소스 — 스타일시트, CSS 커스텀 속성, Tailwind 설정, 토큰 파일, 또는 컴포넌트 스타일 — 에서 도출됩니다. 제3자 브랜드의 `DESIGN.md`를 결코 붙여 넣지 않으며, 참조 카탈로그는 구조를 위한 영감일 뿐 결코 내용을 위한 것이 아닙니다.
- **조정하라, 덮어쓰지 마라.** 기존 `DESIGN.md`나 토큰 소스는 덮어쓰지 않고 가산적으로 조정됩니다. 파괴적 변경에는 승인이 필요합니다.
- **참조에 의한 발견.** `DESIGN.md`가 어디에 있든 `AGENTS.md`(그리고 `CLAUDE.md`)가 그것을 참조합니다 — 에이전트가 그것을 로드하도록 보장하는 것은 물리적 위치가 아니라 그 포인터입니다.
- **실용적이며, 경직되게 묶이지 않음.** 떠오르는 `DESIGN.md` 관례를 따를 형태로 참조하되, Markdown 우선을 유지하며 어떤 단일 토큰 스키마에도 묶이지 않습니다.

## 프런트엔드 범위, 감지 시 기본 활성

이 애드온은 실제 UI 표면을 가진 리포지토리 — CSS 커스텀 속성이 있는 스타일시트, Tailwind 설정이나 `@theme` 블록, UI 컴포넌트, 또는 브랜드/스타일 가이드 — 를 위한 것입니다. 백엔드, CLI, 또는 라이브러리 전용 리포지토리에는 **결코** 제안되지 않습니다. UI 표면이 **감지되면** 온보딩은 이를 **기본 활성**으로 다룹니다. 신뢰 모드에서는 애드온을 적용하고, 가이드 모드에서는 강력히 권장합니다. 결코 필수는 아닙니다 — 애드온이 하나도 없는 리포지토리도 완전히 적합하며, 언제든 거절할 수 있습니다.

## 선택적 명령

채택되면, 애드온은 나중에 `DESIGN.md`를 재생성하거나 갱신할 수 있도록 리포지토리의 `.agents/commands/`에 `/design-system` 위임자를 설치할 수 있습니다. 명령 설치는 선택적이며, 거절된 애드온은 아무것도 설치하지 않습니다.

## 기능별 디자인 문서와의 관계

이것은 **리포지토리 수준의 영속적인** 디자인 시스템 파일입니다 — 도구에 묶인 스펙 주도 워크플로의 기능별 기술 디자인 문서("요구사항 → 디자인 → 작업"의 `design.md`)와는 구별됩니다. Deep Work Plan은 의도적으로 별도의 기능별 디자인 문서 아키타입을 제공하지 않습니다. 계획의 README, 각 작업의 인수 기준, 그리고 검증 게이트가 이미 그 역할을 담당합니다. 이 애드온은 그 역할이 채우지 못하는 한 가지 공백을 메웁니다: 지속적이고 리포지토리 고유의 UI 디자인 컨텍스트.
