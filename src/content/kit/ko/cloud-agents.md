# 클라우드 에이전트 어댑터

클라우드 및 백그라운드 에이전트 — Claude Code 원격 작업, OpenAI Codex 클라우드, Jules 클래스 에이전트, 그리고 유사한 비동기 실행 환경 — 는 설계상 일시적입니다: 세션이 시작되고, 제한된 시간 동안 실행되며, 종료됩니다. DWP는 계획을 세션이 아닌 내구성 있는 산출물로 만들어 그 제약을 기능으로 전환합니다.

## 핵심 통찰

리포지토리의 `.dwp/` 디렉터리가 **내구성 있는 스펙 및 상태 레이어**입니다. 클라우드 세션은 **일시적 실행자**입니다. 계획이 실행자에게 무엇을 해야 할지 알려주고; `state.json`이 어디서 재개해야 할지 알려줍니다. 세션은 무엇이든 될 수 있습니다 — 다른 모델, 다른 공급자, 새 컨테이너 — 그리고 계획은 멈춘 곳에서 정확히 계속됩니다.

## 지원 수준

**Full** — 리포지토리를 읽고, `AGENTS.md`를 따르며, 셸 명령을 실행할 수 있는 모든 클라우드 에이전트는 플랫폼별 어댑터 없이 DWP를 지원합니다.

## 깨어남-재개-양보 루프

각 클라우드 에이전트 턴은 다음 패턴을 따릅니다.

1. **깨어남** — 세션이 시작됩니다(크론 일정, CI 이벤트, 또는 사용자 작업에 의해 트리거됨).
2. **DWP 재개 프로토콜 실행** — 계획 README를 읽고, 체크포인트를 찾고, `state.json`을 Markdown과 조정하고, 이음새를 점검하고, 스모크 테스트를 실행합니다. 참고: [DWP 스펙](/spec/dwp-specification#the-dwp-resume-protocol).
3. **차단 상태 확인** — `state.json.blocked`가 설정되어 있으면, 이유를 보고하고 실행하지 않고 양보합니다.
4. **다음 원자적 작업 실행** — 하나의 작업을 전체 검증 게이트를 통과하여 실행합니다.
5. **상태 레이어 갱신** — 새 작업 상태, 게이트 기록, 결과 기록, 커밋 해시로 `state.json`을 원자적으로 재작성합니다.
6. **양보** — 세션이 깔끔하게 종료됩니다; 다음 턴이 갱신된 상태에서 이어받습니다.

## 요구 사항

- 계획은 기계 가독 상태 레이어(`manifest.json` + `state.json`)를 담아야(MUST) 합니다. 없으면 일시적 세션이 재개 위치를 신뢰할 수 있게 결정할 방법이 없습니다. 참고: [계획 상태](/spec/plan-state).
- 계획은 무인 클라우드 턴 전에 사람이 승인해야(MUST) 합니다. 계획 승인이 제어 지점입니다.
- 에이전트는 제한된 권한 하에 운영됩니다: 범위를 확장해서는 안 되고(MUST NOT), 계획이 명시적으로 승인하지 않은 파괴적이거나 외부를 향하는 작업을 수행해서는 안 되며(MUST NOT), 발견되었지만 계획에 없는 작업을 현재 작업 지시 범위를 넘어 즉흥적으로 처리해서는 안 됩니다(MUST NOT). 참고: [에이전트 프로토콜](/spec/agent-protocol#execution-profiles).

## 참고

클라우드 에이전트 세션은 무인 실행 프로필의 극단적 경우입니다: 세션 연속성 없음, 최대 계획 내구성. DWP의 `.dwp/` 디렉터리는 이를 작동하게 하도록 특별히 설계되었습니다 — 에이전트가 작업 시작, 계속, 완료에 필요한 모든 상태가 모든 세션이 새 체크아웃에서 읽을 수 있는 일반 Markdown과 JSON 파일에 있습니다.
