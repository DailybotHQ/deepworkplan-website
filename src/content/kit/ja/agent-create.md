# agent-create

現在のリポジトリに新しいエージェントを作成します。Deep Work Plan の **author** サブスキルへの薄い委譲です。

## 何をするか

`agent-create` は author サブスキルのエージェント作成フローへ振り分けます。エージェントは、ある種類のタスクを扱うために派遣される専門化されたワーカーの定義（役割、モデル階層、ツール、システムプロンプト）です。「誰が X を行うか」を符号化し、`.agents/agents/<name>.md` の配下に存在します。

## 使い方

```
/agent-create
```

## 振る舞い

1. リポジトリの `.agents/` レイアウトとローカルの規約を検出する。
2. 固有のモデルやツールを必要とする繰り返し現れる役割を確認し、ケバブケースで名づける。
3. エージェント（Role、Inputs、Process、Output、エスカレーション）を整備し記入する。
4. 正当化されたモデル階層、すなわち light、standard、heavy を、ベンダーのモデル ID をハードコードせずに選ぶ。
5. `.agents/docs/` のカタログを一致するよう更新する。

## 補足

手順には `/skill-create` を使います。モデル階層は抽象的に保ち、リポジトリの設定で対応づけます。
