# Додаток AI Diff Reviewer

Підключає виконання Deep Work Plan до **[AI Diff Reviewer](https://github.com/DailybotHQ/ai-diff-reviewer)** (опублікований у маркетплейсі як **«AI Diff Reviewer»**, поточна версія **v2.0.0**), щоб обов'язкове фінальне завдання **Огляду безпеки** отримало структурований локальний огляд — вердикт, таблицю знахідок і рівень серйозності — а при виборі Flow B кожен pull request може бути заблокований тим самим оглядом у CI. Необов'язковий **додаток**; ніколи не потрібен для відповідності AI-first.

Основна методологія Deep Work Plan має **нульову** залежність від AI Diff Reviewer. Репозиторій без додатків повністю відповідає вимогам. Пропонуйте цей додаток лише тоді, коли розробник або команда прагне структурованої якості огляду; ніколи не встановлюйте автоматично для всіх. Завжди запитуйте Flow A чи Flow B — ніколи не передбачайте значення за замовчуванням.

## Коли використовувати

| Сигнал | Дія |
|--------|--------|
| Команда хоче CI-шлюз для злиття PR зі структурованими знахідками | **Рекомендувати Flow B** під час онбординг |
| Особистий або експериментальний репозиторій; локального огляду перед push достатньо | **Запропонувати Flow A** |
| Немає потреби в додатковій поверхні огляду | **Пропустити** — базовий Огляд безпеки діє |

## Два потоки прийняття

| Потік | Що ви отримуєте |
|------|----------------|
| **A — лише локально** | Вбудований skill + обов'язковий `.review/extension.md` (через `generate-extension`). Розширює Огляд безпеки локальним проходом. Без робочого процесу GitHub Actions. |
| **B — подвійна поверхня** | Flow A плюс `setup` записує `.github/workflows/pr-review.yml` (Action `@v2`), той самий файл розширення для локального та CI. Необов'язковий супутник `apply-review` після того, як CI публікує знахідки. |

Виявлення для розширення Огляду безпеки потребує **skill + файл розширення** в одному з: `.review/extension.md`, `.github/ai-diff-reviewer/extension.md` або `.github/ai-pr-reviewer/extension.md`. Одного skill недостатньо.

## Що підключає цей додаток (навмисно обмежено)

Додаток DWP **не** перевинаходить рецензента. Він делегує встановлення, методологію, майстер CI, написання розширень, складання PR і перегляд після CI п'ятьом підскілам вихідного skill (батьківський типовий потік, `generate-extension`, `setup`, `open-pr`, `apply-review`).

### Розширення Огляду безпеки

При виявленні `create` / `execute` додають крок локального огляду до обов'язкового завдання Огляду безпеки. Вивід додається під `## AI Diff Reviewer local review` у `analysis_results/SECURITY_REVIEW.md`.

- **М'який збій (лише виклик):** відсутній skill, відсутнє розширення або помилка виклику → попередити один раз і продовжити; ніколи не провалювати завдання через це.
- **Шлюз після завершеного проходу:** знахідки `critical` продовжують блокувати завершення Огляду безпеки до виправлення або явного прийняття (існуючий SR-контракт). `warning` / `info` документуються, але не блокують.
- **Flow A не потребує CI-секрету.** Незаданий `CURSOR_API_KEY` не має пригнічувати локальний прохід.

### CI-шлюз Flow B (необов'язково)

Закріплений Action `DailybotHQ/ai-diff-reviewer@v2`, зазвичай заблокований міткою (`ready`), зі стабільно названим завданням **AI review gate** для захисту гілки та необов'язковою міткою пропуску `skip-review-label: skip-ai-review`. Спільний `prompt.md` + розширення вирівнює методологію та серйозність; при ітераційно-усвідомленому огляді раунди CI 2+ можуть бути коротшими, тоді як локальний прохід залишається повним.

### Необов'язковий супутник `apply-review`

Після того, як CI публікує огляд, розробник може викликати `apply-review` під час `execute`, щоб переглянути знахідки по одній (застосувати / відкласти / пропустити) зі згоди. За замовчуванням лише читання; ніколи не є файлом завдання плану (порушить обов'язковий порядок фінальних завдань).

## Поведінка

- **Запитуйте про потік; ніколи не вгадуйте.** Встановлення незапитаного робочого процесу має більший слід, ніж залишатися на Flow A.
- **Узгоджуйте, не перезаписуйте.** Існуючий skill, розширення або `pr-review.yml` зберігаються; заповнюйте лише прогалини.
- **Автентифікація відкладена.** Секрети провайдера для CI налаштовує супроводжувач; цей додаток ніколи не зберігає облікові дані.
- **Нейтральний щодо провайдера.** Відмова залишає повністю AI-first репозиторій.

## Примітки

Необов'язковий і ніколи не потрібний. Вихідний skill: [DailybotHQ/ai-diff-reviewer](https://github.com/DailybotHQ/ai-diff-reviewer). Сторінка специфікації: [Add-ons](/spec/addons).
