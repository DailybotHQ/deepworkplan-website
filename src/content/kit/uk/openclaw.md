# Адаптер OpenClaw

OpenClaw — автономна агентна платформа, що запускає довготривалі робочі простори із запланованими поворотами. Вона нативно сканує `<workspace>/.agents/skills/` для встановлених пакетів скілів, що означає завантаження скіла DWP без будь-якого проміжного адаптера — встановіть його там, і OpenClaw підхопить його автоматично.

## Рівень підтримки

**Повний** — OpenClaw читає `AGENTS.md` та завантажує скіл через стандарт AgentSkills. Доступні всі суб-скіли DWP (`onboard`, `create`, `execute`, `refine`, `resume`, `status`, `verify`).

## Встановлення

```bash
openclaw skills install deepworkplan
```

Або вручну: клонуйте `DailybotHQ/deepworkplan-skill` у `<workspace>/.agents/skills/deepworkplan/`.

## Виклик

Простий текст — OpenClaw не використовує префікс слеш-команди:

```
onboard this workspace with deepworkplan
create a plan for <goal>
resume the open plan
```

## Відображення автономного виконання

Примітиви планування OpenClaw безпосередньо відображаються на модель продовження DWP:

| Примітив OpenClaw | Роль DWP |
|--------------------|----------|
| Корінь робочого простору | Архетип [робочого простору агента](/spec/archetypes): `AGENTS.md`, `.agents/`, `.dwp/` у корені робочого простору |
| `<workspace>/.agents/skills/` (нативне сканування рівня 2) | Де живе цей пакет — проміжний адаптер не потрібен |
| Heartbeat або cron-поворот | Один поворот [запланованого продовження](/spec/agent-protocol#scheduled-continuation): прокидання → Протокол відновлення DWP → наступне атомарне завдання → оновлення стану → поступка |
| Перевірки `HEARTBEAT.md` | Додайте один рядок: "Якщо `.dwp/plans/` має відкритий план, відновіть його на одне завдання." |
| Постійні накази | Межа схвалення плану та обмежені повноваження автономного профілю |

Операційно:

1. Людина інтерактивно створює та схвалює план. Схвалення є контрольною точкою — автономні повороти ніколи не створюють і не виконують за один поворот.
2. План MUST нести машиночитаний рівень стану (`manifest.json` + `state.json`). У робочому просторі без git `state.json` є тим, що робить відновлення після збоїв можливим. Див. [Стан плану](/spec/plan-state).
3. Кожен heartbeat або cron-поворот виконує щонайбільше одне завдання, проходить свій валідаційний gate, атомарно оновлює `state.json` та поступається.
4. У разі будь-якої умови зупинки агент записує поле `blocked` у `state.json` та звітує через нормальний канал робочого простору. Наступний людський погляд — або звіт heartbeat — бачить рівно те, що потрібно.

Результат: багатоденний план, що виживає після перезапусків, змін моделі та меж сесій, виконаний уночі демоном — із тими самими gate, які мало б виконання під наглядом людини.

## Нотатки

Робочий простір OpenClaw є канонічним прикладом архетипу [робочого простору агента](/spec/archetypes). Інтеграція `HEARTBEAT.md` та вимога `state.json` пакету скілів DWP були розроблені з урахуванням цієї платформи.
