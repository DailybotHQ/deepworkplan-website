# dwp-verify

Obiektywnie sprawdź zgodność repozytorium i planu — tylko do odczytu.

<CmdVerify lang="pl" class="not-prose my-8" />

## Co robi

`dwp-verify` sprawdza mechanicznie, czy repozytorium jest AI-first (zgodne z DeepWorkPlan) i czy jego plany są poprawnie sformułowane, a następnie zgłasza wynik pass/fail. Niczego nie zmienia.

## Użycie

```
/dwp-verify            # verify the repository
/dwp-verify plan auth  # also verify a specific plan
/dwp-verify all        # repository + every plan
```

## Zachowanie

1. Sprawdzenie repozytorium: `AGENTS.md` z rzeczywistymi, wykonywalnymi komendami; rozwiązanie `CLAUDE.md`; kategorie `docs/`; katalog `.agents/` zgodny z dyskiem; cienkie delegatory `dwp-*`; pomijane przez git `.dwp/` i `tmp/`; rozwiązywalny skill.
2. Sprawdzenie planu (na żądanie): każde zadanie ma kryteria akceptacji i bramkę walidacji, postęp jest zapisywany, obecne są trzy obowiązkowe zadania końcowe (w tym Security Review), a zadania na nowo odnoszą się do celu.
3. Zgłoszenie **CONFORMANT** lub **NOT CONFORMANT** wraz z wymienieniem każdego niepowodzenia.
4. Zaproponowanie ujęcia braków w postaci Deep Work Plan; nigdy nie naprawia po cichu.

## Aliasy
