# dwp-verify

客观地核查仓库与计划的符合性——只读。

<CmdVerify lang="en" class="not-prose my-8" />

## 它做了什么

`dwp-verify` 以机械方式核查一个仓库是否 AI-first（符合 DeepWorkPlan），以及它的计划是否结构良好，然后报告通过/未通过。它不改动任何东西。

## 用法

```
/dwp-verify            # verify the repository
/dwp-verify plan auth  # also verify a specific plan
/dwp-verify all        # repository + every plan
```

## 行为

1. 核查仓库：带有真实、可运行命令的 `AGENTS.md`；`CLAUDE.md` 的解析；`docs/` 的各类别；与磁盘一致的 `.agents/` 目录；轻量的 `dwp-*` 委派器；被 gitignore 的 `.dwp/` 与 `tmp/`；一个可解析的技能。
2. 核查一份计划（在被请求时）：每项任务都有验收标准与验证关卡，进展已被持久化，两项强制收尾任务均已就位，并且任务重新锚定到目标。
3. 报告 **CONFORMANT** 或 **NOT CONFORMANT**，并列出每一处失败。
4. 提议把缺口捕获为一份 Deep Work Plan；绝不悄悄修复。

## 别名

- `dwp:verify`
