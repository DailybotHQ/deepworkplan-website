# เทมเพลตแผนและงาน

Deep Work Plan คือไดเรกทอรีของไฟล์ markdown รูปร่างของไดเรกทอรีนั้น — และของแต่ละไฟล์ภายใน — คือสิ่งที่ทำให้แผนทบทวนได้โดยทั้งมนุษย์และเอเจนต์

## โครงสร้างแผน

<PlanTree lang="th" class="not-prose my-10" />

แผนคือไดเรกทอรีภายใต้ `.dwp/plans/` ที่ตั้งชื่อว่า `PLAN_<slug>/` ภายในประกอบด้วย

- `README.md` — ภาพรวมของแผน ได้แก่ เป้าหมาย วัสดุต้นทาง ตารางงาน และสถานะ
- หนึ่งไฟล์ต่อหนึ่งงาน ตั้งชื่อว่า `<n>.task_<slug>.md`
- `PROGRESS.md` — log การทำงานที่บันทึกต่อเนื่อง

## โครงสร้างไฟล์งาน

ไฟล์งานแต่ละไฟล์ตั้งชื่อว่า `<n>.task_<slug>.md` และเป็นไปตามโครงสร้างเก้าส่วน ได้แก่ เป้าหมาย บริบท ขั้นตอน เกณฑ์การยอมรับ การตรวจสอบ ไฟล์ ส่วนพึ่งพิง ความเสี่ยง และการเสร็จสิ้นและ Log ส่วนต่าง ๆ ปรากฏตามลำดับนั้นเสมอ เพื่อให้ผู้อ่านทุกคนรู้ว่าควรมองหาที่ใด แต่ละงานยึดโยงกลับไปยังเป้าหมายของแผนก่อนลงมือ ซึ่งช่วยกันไม่ให้เอเจนต์เบี่ยงเบนไปตลอดช่วงเวลายาวนานหลายชั่วโมง

## สามงานสุดท้ายที่บังคับ

ทุกแผนต้องจบด้วยสามงานมาตรฐาน งานเหล่านี้รับประกันว่างานได้รับการตรวจสอบ ความรู้ถูกเก็บรวบรวม และผู้มีส่วนได้ส่วนเสียได้รับข้อมูล

1. **การทบทวนความปลอดภัย (Security Review)** — ตรวจสอบทุกสิ่งที่แผนเปลี่ยนแปลงเพื่อหาความลับ ความเสี่ยงจาก injection และพื้นผิวการโจมตีใหม่ และคง `docs/SECURITY.md` ให้เป็นปัจจุบัน ความปลอดภัยไม่ใช่สายงานแยกที่ถูกแปะเพิ่มตอนท้าย ทุกแผนทิ้งการเปลี่ยนแปลงของตัวเองไว้ในสภาพที่ผ่านการตรวจสอบแล้ว และสิ่งที่พบในระดับวิกฤตจะปิดกั้นการเสร็จสิ้น
2. **การค้นพบ Skills และ Agents** — ระบุ skill และ agent ที่นำกลับมาใช้ใหม่ได้ซึ่งถูกสร้างขึ้นระหว่างแผน เพื่อให้งานทบต้นทบดอกไปไกลกว่าความพยายามครั้งเดียวนี้
3. **รายงานสรุปสำหรับผู้บริหาร** — บทสรุปผลลัพธ์ที่กระชับซึ่งเขียนสำหรับผู้มีส่วนได้ส่วนเสีย โดยแยกแยะว่าสิ่งใดถูกส่งมอบแล้วและสิ่งใดยังคงเหลืออยู่

## ข้อตกลงของ README และ PROGRESS

`README.md` ของแผนต้องมีชื่อเรื่อง (`# Deep Work Plan: <name>`) คำบอกกล่าวเป้าหมายแบบร้อยแก้ว ส่วนวัสดุต้นทางที่เป็นทางเลือก ตารางงาน (หมายเลข ชื่องาน ช่องทำเครื่องหมายสถานะ) และบรรทัดสถานะแผนในรูปแบบ `<n>/<total> tasks complete`

`PROGRESS.md` คือ log การทำงานแบบเพิ่มได้อย่างเดียว แต่ละรายการบันทึก timestamp แบบ ISO 8601 หมายเลขและชื่องาน สิ่งที่ทำไป และการเบี่ยงเบนหรือเหตุผลในการข้ามใด ๆ เพราะมันมีแต่จะเติบโตขึ้นเรื่อย ๆ มันจึงเป็นประวัติที่เชื่อถือได้ว่าแผนคลี่คลายไปอย่างไรจริง ๆ

## หัวข้อและการอ้างอิงข้าม

หัวข้อทั้งหมดใช้ตัวพิมพ์แบบประโยค และเอกสารหลีกเลี่ยงภาษาเชิงการตลาดและเครื่องหมายอัศเจรีย์ ข้อตกลงเหล่านี้ทำให้แผนสอดคล้องกันข้ามเอเจนต์และตลอดเวลา
