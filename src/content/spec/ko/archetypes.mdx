# 아키타입

**버전 1.1.** DWP는 세 가지 아키타입을 인식합니다. 아키타입은 에이전트가 온보딩하는 방식, 계획의 범위가 정해지는 방식, 그리고 기계 가독 상태 레이어가 필요한지 여부를 결정합니다.

> **v1.1에서 추가.** **에이전트 작업 공간**(§3)이 세 번째 아키타입으로 합류: 자율 에이전트의 장기 실행 홈. Git은 필수 가정이 아닌 RECOMMENDED가 됩니다. 두 개의 v1.0 아키타입은 변경되지 않습니다.

## 개별 리포지토리

자기 완결적 코드베이스 — 애플리케이션, 라이브러리, 또는 서비스. 계획은 코드에 직접 작용합니다.

특성:

- 하나의 일관된 코드베이스.
- 계획이 이 리포지토리의 파일을 수정합니다.
- 리포지토리 루트의 `.dwp/` 작업 공간.

## 오케스트레이터 허브

여러 하위 리포지토리를 관리하는 조율 리포지토리. 계획이 하위 리포지토리에서 하위 계획을 생성할 수 있습니다.

특성:

- 여러 하위 리포지토리를 조율합니다.
- 계획이 하위 계획에 위임할 수 있습니다.
- 관리 대상 리포지토리의 레지스트리를 유지합니다.
- 허브 루트의 `.dwp/` 작업 공간이 리포지토리 간 상태를 추적합니다.

## 에이전트 작업 공간

에이전트 작업 공간은 자율 에이전트의 장기 실행 홈입니다 — OpenClaw 작업 공간, Hermes 서비스 디렉터리, 개인 비서 데몬의 데이터 디렉터리, 또는 클라우드 에이전트의 영속 볼륨. 이것은 반드시 git 리포지토리가 아닌 작업 공간이며, 그 주요 산출물은 하나의 코드베이스가 아닌 에이전트의 진행 중인 작업입니다.

이 아키타입을 가능하게 하는 통찰: **하니스는 리포지토리가 아닌 작업 공간입니다.** 방법론이 리포지토리를 위해 정의하는 모든 하니스 요소에는 직접 작업 공간 등가물이 있습니다.

| 하니스 요소(리포지토리) | 에이전트 작업 공간 등가물 |
|------------------------------|---------------------------|
| `AGENTS.md`(규칙, 빠른 명령) | 작업 공간의 상시 컨텍스트 — `AGENTS.md` 자체, 또는 플랫폼의 상시 지시 파일 |
| `docs/`(내구성 있는 지식) | 작업 공간 지식 파일과 메모리 문서 |
| `.agents/`(스킬, 에이전트, 명령) | 플랫폼의 스킬 디렉터리 — OpenClaw는 네이티브로 `<workspace>/.agents/skills/` 스캔 |
| `.dwp/`(계획, 초안) | 작업 공간 루트의 `.dwp/` — 변경 없음 |
| git 로그(상태, 재개 가능성) | 계획별 `state.json`([계획 상태](/spec/plan-state)), 여기서 REQUIRED |

에이전트 작업 공간은 작업 공간 루트에 `AGENTS.md`, `.agents/`, `.dwp/`를 제공해야(MUST) 합니다.

**Git은 RECOMMENDED이지, REQUIRED가 아닙니다.** git이 없는 경우, 모든 계획은 기계 가독 상태 레이어를 담아야(MUST) 합니다: `state.json`의 체크포인트, 게이트 기록, 작업별 타임스탬프가 리포지토리에서 git 로그가 담당하는 복구 정보를 담습니다.

에이전트 작업 공간의 계획은 일반적으로 무인으로 실행됩니다: 예약된 하트비트 또는 크론 턴이 DWP 재개 프로토콜을 통해 열린 계획을 재개하고, 다음 원자적 작업을 실행하고, 상태 레이어를 갱신하고, 양보합니다. 세션이 아닌 계획이 연속성의 단위입니다.

## 분류 휴리스틱

<ArchetypesTree lang="ko" class="not-prose my-10" />

**에이전트 작업 공간**으로 먼저 분류합니다. 대상이 자율 에이전트 플랫폼의 작업 디렉터리일 때 — 신호: 플랫폼 식별 파일(OpenClaw의 `SOUL.md` 또는 `HEARTBEAT.md` 등), 주요 애플리케이션 스택 없음, 주로 에이전트 자체 상태인 콘텐츠. 하나의 강력한 플랫폼 신호로 충분합니다.

그 외에, 다음의 명확한 대다수가 해당할 때 **오케스트레이터 허브**로 분류합니다: 여러 중첩된 git 리포지토리 또는 서브모듈; 관리 대상 리포지토리의 레지스트리 또는 매니페스트; 외부 리포지토리를 가리키는 설정. 그렇지 않으면 **개별 리포지토리**로 분류합니다 — 안전한 기본값입니다.

신호가 모호할 때, 에이전트는 평가와 증거를 사용자에게 제시하고 진행하기 전에 확인을 요청해야(MUST) 합니다.

## 온보딩 차이

| 측면 | 개별 | 오케스트레이터 | 에이전트 작업 공간 |
|--------|------------|--------------|-----------------|
| 범위 | 이 리포지토리 | 여러 리포지토리 | 작업 공간과 그 계획들 |
| 온보딩 | 리포지토리 구조 | 허브 레지스트리 | 플랫폼 파일 + 작업 공간 관례 |
| 계획 대상 | 로컬 파일 | 하위 계획 | 로컬 또는 외부 리포지토리 |
| 상태 | 로컬 `.dwp/` | 리포지토리 간 `.dwp/` | `.dwp/` + `state.json`(git 없이 REQUIRED) |
| Git | 필수 | 필수 | RECOMMENDED |
| 상태 레이어 | RECOMMENDED | RECOMMENDED | git 없을 때 REQUIRED; 무인 실행 시 REQUIRED |
