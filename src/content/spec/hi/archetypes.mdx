# आर्किटाइप

**संस्करण 1.1.** DWP तीन आर्किटाइप पहचानता है। आर्किटाइप यह तय करता है कि एक एजेंट कैसे ऑनबोर्ड करता है, योजनाओं का दायरा कैसे बँधता है, और मशीन-पठनीय स्थिति परत आवश्यक है या नहीं।

> **v1.1 में योगात्मक।** **एजेंट कार्यस्थल** (§3) तीसरे आर्किटाइप के रूप में सम्मिलित होता है: एक स्वायत्त एजेंट का दीर्घकालिक घर। Git को मान लेने के बजाय RECOMMENDED बनाया गया है। दोनों v1.0 आर्किटाइप अपरिवर्तित हैं।

## स्वतंत्र रिपॉज़िटरी

एक स्व-निहित कोडबेस — एक एप्लिकेशन, एक लाइब्रेरी, या एक सेवा। योजनाएँ सीधे कोड पर काम करती हैं।

विशेषताएँ:

- एकल सुसंगत कोडबेस।
- योजनाएँ इस रिपॉज़िटरी की फ़ाइलें संशोधित करती हैं।
- रिपॉज़िटरी रूट पर `.dwp/` कार्यस्थल।

## ऑर्केस्ट्रेटर हब

एक समन्वय रिपॉज़िटरी जो कई चाइल्ड रिपॉज़िटरीज़ का प्रबंधन करती है। योजनाएँ सब-रिपॉज़िटरीज़ में चाइल्ड योजनाएँ बना सकती हैं।

विशेषताएँ:

- कई सब-रिपॉज़िटरीज़ का समन्वय करती है।
- योजनाएँ चाइल्ड योजनाओं को सौंप सकती हैं।
- प्रबंधित रिपॉज़िटरीज़ की एक रजिस्ट्री बनाए रखती है।
- हब रूट पर `.dwp/` कार्यस्थल क्रॉस-रिपॉज़िटरी स्थिति को ट्रैक करता है।

## एजेंट कार्यस्थल

एक एजेंट कार्यस्थल एक स्वायत्त एजेंट का दीर्घकालिक कार्यशील घर है — एक OpenClaw workspace, एक Hermes service directory, एक personal-assistant daemon की data directory, या एक cloud agent का persistent volume। यह एक कार्यस्थल है, ज़रूरी नहीं कि एक git रिपॉज़िटरी हो, और इसका प्राथमिक उत्पाद एक codebase के बजाय एजेंट का चल रहा काम है।

वह अंतर्दृष्टि जो इस आर्किटाइप को संभव बनाती है: **हार्नेस एक कार्यस्थल है, विशेष रूप से एक रिपॉज़िटरी नहीं।** पद्धति एक रिपॉज़िटरी के लिए जो हर हार्नेस तत्व परिभाषित करती है उसका एक सीधा कार्यस्थल समकक्ष है:

| हार्नेस तत्व (रिपॉज़िटरी) | एजेंट कार्यस्थल समकक्ष |
|------------------------------|---------------------------|
| `AGENTS.md` (नियम, त्वरित कमांड) | workspace का standing context — `AGENTS.md` स्वयं, या प्लेटफ़ॉर्म की standing-orders फ़ाइल |
| `docs/` (टिकाऊ ज्ञान) | Workspace knowledge फ़ाइलें और memory दस्तावेज़ |
| `.agents/` (skills, agents, commands) | प्लेटफ़ॉर्म की skill directory — OpenClaw नेटिव रूप से `<workspace>/.agents/skills/` स्कैन करता है |
| `.dwp/` (plans, drafts) | workspace रूट पर `.dwp/` — अपरिवर्तित |
| git log (state, resumability) | प्रति-योजना `state.json` ([योजना-स्थिति](/spec/plan-state)), यहाँ REQUIRED |

एक एजेंट कार्यस्थल में MUST workspace रूट पर `AGENTS.md`, `.agents/`, और `.dwp/` होने चाहिए।

**Git RECOMMENDED है, REQUIRED नहीं।** जहाँ git अनुपस्थित हो, हर योजना में MUST मशीन-पठनीय स्थिति परत होनी चाहिए: `state.json` के checkpoint, गेट रिकॉर्ड, और प्रति-कार्य timestamps वह पुनर्प्राप्ति जानकारी वहन करते हैं जो git लॉग एक रिपॉज़िटरी में वहन करता है।

एजेंट कार्यस्थल में योजनाएँ आमतौर पर अनुपस्थित चलती हैं: एक अनुसूचित heartbeat या cron turn DWP Resume Protocol के माध्यम से खुली योजना फिर शुरू करता है, अगला परमाणु कार्य निष्पादित करता है, स्थिति परत अद्यतन करता है, और yield करता है। योजना — सत्र नहीं — निरंतरता की इकाई है।

## वर्गीकरण अनुमान-नियम

<ArchetypesTree lang="hi" class="not-prose my-10" />

पहले **एजेंट कार्यस्थल** के रूप में वर्गीकृत करें, जब लक्ष्य एक स्वायत्त एजेंट प्लेटफ़ॉर्म की कार्यशील directory हो — संकेत: एक प्लेटफ़ॉर्म पहचान फ़ाइल (जैसे OpenClaw की `SOUL.md` या `HEARTBEAT.md`), कोई प्राथमिक application stack नहीं, और सामग्री जो मुख्यतः एजेंट की अपनी state है। कोई भी एक मज़बूत प्लेटफ़ॉर्म संकेत पर्याप्त है।

अन्यथा, **ऑर्केस्ट्रेटर हब** के रूप में वर्गीकृत करें जब निम्नलिखित का स्पष्ट बहुमत हो: कई नेस्टेड git रिपॉज़िटरीज़ या submodules; प्रबंधित रिपॉज़िटरीज़ की एक registry या manifest; बाहरी रिपॉज़िटरीज़ की ओर इंगित करती configuration। अन्यथा **स्वतंत्र रिपॉज़िटरी** के रूप में वर्गीकृत करें — यह सुरक्षित डिफ़ॉल्ट है।

जब संकेत अस्पष्ट हों, एजेंट को MUST उपयोगकर्ता को अपना आकलन और साक्ष्य प्रस्तुत करना चाहिए और आगे बढ़ने से पहले पुष्टि माँगनी चाहिए।

| पहलू | स्वतंत्र | ऑर्केस्ट्रेटर | एजेंट कार्यस्थल |

| दायरा | यह रिपॉज़िटरी | कई रिपॉज़िटरीज़ | workspace और उसकी योजनाएँ |
| ऑनबोर्डिंग | रिपॉज़िटरी संरचना | हब रजिस्ट्री | प्लेटफ़ॉर्म फ़ाइलें + workspace परिपाटियाँ |
| योजना लक्ष्य | स्थानीय फ़ाइलें | चाइल्ड योजनाएँ | स्थानीय या बाहरी रिपॉज़िटरीज़ |
