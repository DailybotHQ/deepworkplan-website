# Các kiểu hình

**Phiên bản 1.0.** DWP nhận diện hai kiểu hình repository. Kiểu hình quyết định cách một agent khởi tạo và cách các kế hoạch được giới hạn phạm vi.

## Repository độc lập

Một codebase khép kín — một ứng dụng, một thư viện, hay một dịch vụ. Các kế hoạch tác động trực tiếp lên mã.

Đặc điểm:

- Một codebase mạch lạc duy nhất.
- Các kế hoạch sửa đổi tệp trong repository này.
- Không gian làm việc `.dwp/` tại gốc repository.

## Trung tâm điều phối

Một repository điều phối quản lý nhiều repository con. Các kế hoạch có thể sinh ra các kế hoạch con trong các sub-repository.

Đặc điểm:

- Điều phối nhiều sub-repository.
- Các kế hoạch có thể ủy thác cho các kế hoạch con.
- Duy trì một danh bạ các repository được quản lý.
- Không gian làm việc `.dwp/` tại gốc trung tâm theo dõi trạng thái xuyên repository.

## Phép suy đoán phân loại

<ArchetypesTree lang="vi" class="not-prose my-10" />

Một agent NÊN (SHOULD) phân loại một repository là trung tâm điều phối nếu nó tìm thấy:

- Nhiều repository git lồng nhau hoặc submodule.
- Một danh bạ hay manifest các repository được quản lý.
- Cấu hình trỏ tới các repository bên ngoài.

Nếu không, hãy coi nó là một repository độc lập.

## Khác biệt khi khởi tạo

| Khía cạnh | Độc lập | Điều phối |
|--------|------------|--------------|
| Phạm vi | Repository này | Nhiều repository |
| Khởi tạo | Cấu trúc repository | Danh bạ trung tâm |
| Đích kế hoạch | Tệp cục bộ | Các kế hoạch con |
| Trạng thái | `.dwp/` cục bộ | `.dwp/` xuyên repository |
