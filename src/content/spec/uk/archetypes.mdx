# Архетипи

**Версія 1.1.** DWP розрізняє три архетипи. Архетип визначає, як агент проходить онбординг, як окреслюється обсяг планів та чи є обов'язковим машиночитаний рівень стану.

> **Адитивно у v1.1.** **Робочий простір агента** (§3) приєднується як третій архетип: довготривала домівка автономного агента. Git стає RECOMMENDED, а не обов'язковим. Два архетипи v1.0 незмінні.

## Окремий репозиторій

Самодостатня кодова база — застосунок, бібліотека чи сервіс. Плани діють безпосередньо на код.

Ознаки:

- Єдина цілісна кодова база.
- Плани змінюють файли в цьому репозиторії.
- Робочий простір `.dwp/` у корені репозиторію.

## Хаб-оркестратор

Координаційний репозиторій, що керує кількома дочірніми репозиторіями. Плани можуть породжувати дочірні плани в суб-репозиторіях.

Ознаки:

- Координує кілька суб-репозиторіїв.
- Плани можуть делегувати дочірнім планам.
- Веде реєстр керованих репозиторіїв.
- Робочий простір `.dwp/` у корені хабу відстежує стан між репозиторіями.

## Робочий простір агента

Робочий простір агента — це довготривала робоча домівка автономного агента: робочий простір OpenClaw, службовий каталог Hermes, каталог даних демона-персонального-помічника або постійний том хмарного агента. Це простір, а не обов'язково git-репозиторій, і його основним продуктом є поточна робота агента, а не одна кодова база.

Ключова ідея, що робить цей архетип можливим: **harness — це простір, а не конкретно репозиторій.** Кожен елемент harness, що методологія визначає для репозиторію, має прямий еквівалент у просторі:

| Елемент harness (репозиторій) | Еквівалент у робочому просторі агента |
|------------------------------|--------------------------------------|
| `AGENTS.md` (правила, швидкі команди) | Постійний контекст простору — сам `AGENTS.md` або файл постійних наказів платформи |
| `docs/` (стійкі знання) | Файли знань простору та документи пам'яті |
| `.agents/` (скіли, агенти, команди) | Каталог скілів платформи — OpenClaw нативно сканує `<workspace>/.agents/skills/` |
| `.dwp/` (плани, чернетки) | `.dwp/` у корені простору — незмінний |
| журнал git (стан, відновлюваність) | `state.json` на план ([Стан плану](/spec/plan-state)), REQUIRED тут |

Робочий простір агента MUST надавати `AGENTS.md`, `.agents/` та `.dwp/` у корені простору.

**Git є RECOMMENDED, але не REQUIRED.** Де git відсутній, кожен план MUST нести машиночитаний рівень стану: checkpoint `state.json`, записи gate та мітки часу по завданнях несуть інформацію для відновлення, яку несе журнал git у репозиторії.

Плани у робочому просторі агента зазвичай виконуються автономно: запланований heartbeat або cron-поворот відновлює відкритий план через Протокол відновлення DWP, виконує наступне атомарне завдання, оновлює рівень стану та поступається. Одиницею неперервності є план, а не сесія.

## Евристика класифікації

<ArchetypesTree lang="uk" class="not-prose my-10" />

Класифікуйте як **робочий простір агента** спочатку, коли ціль є робочим каталогом автономної агентної платформи — сигнали: файл ідентичності платформи (наприклад, `SOUL.md` або `HEARTBEAT.md` OpenClaw), відсутність основного стеку застосунку та вміст, що переважно є власним станом агента. Достатньо одного сильного сигналу платформи.

В іншому разі класифікуйте як **хаб-оркестратор**, коли чітка більшість із наступного виконується: кілька вкладених git-репозиторіїв чи субмодулів; реєстр або маніфест керованих репозиторіїв; конфігурація, що вказує на зовнішні репозиторії. Класифікуйте як **окремий репозиторій** в інших випадках — це безпечне значення за замовчуванням.

Коли сигнали неоднозначні, агент MUST представити свою оцінку та докази користувачеві й попросити підтвердження перед продовженням.

## Відмінності онбордингу

| Аспект | Окремий | Оркестратор | Робочий простір агента |
|--------|---------|-------------|------------------------|
| Обсяг | Цей репозиторій | Кілька репозиторіїв | Простір та його плани |
| Онбординг | Структура репозиторію | Реєстр хабу | Файли платформи + конвенції простору |
| Ціль плану | Локальні файли | Дочірні плани | Локальні або зовнішні репозиторії |
| Стан | Локальний `.dwp/` | Міжрепозиторний `.dwp/` | `.dwp/` + `state.json` (REQUIRED без git) |
| Git | Required | Required | RECOMMENDED |
| Рівень стану | RECOMMENDED | RECOMMENDED | REQUIRED без git; REQUIRED для автономних запусків |
