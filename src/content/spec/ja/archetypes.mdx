# アーキタイプ

**バージョン 1.0。** DWP は二つのリポジトリアーキタイプを認識します。アーキタイプは、エージェントがどうオンボーディングし、計画のスコープがどう定められるかを決めます。

## 個別リポジトリ

自己完結したコードベース、すなわちアプリケーション、ライブラリ、サービスです。計画はコードに直接作用します。

特徴。

- 単一の一貫したコードベース。
- 計画はこのリポジトリ内のファイルを変更する。
- `.dwp/` 作業領域はリポジトリのルートに。

## オーケストレーターハブ

複数の子リポジトリを管理する調整用リポジトリです。計画はサブリポジトリの中に子計画を生成することがあります。

特徴。

- 複数のサブリポジトリを調整する。
- 計画は子計画へ委譲することがある。
- 管理対象リポジトリのレジストリを維持する。
- ハブのルートにある `.dwp/` 作業領域がリポジトリ横断の状態を追跡する。

## 分類のヒューリスティック

<ArchetypesTree lang="ja" class="not-prose my-10" />

エージェントは、次を見つけた場合、リポジトリをオーケストレーターハブとして分類すべきです（SHOULD）。

- 複数の入れ子になった git リポジトリやサブモジュール。
- 管理対象リポジトリのレジストリやマニフェスト。
- 外部リポジトリを指す設定。

そうでなければ、個別リポジトリとして扱います。

## オンボーディングの違い

| 観点 | 個別 | オーケストレーター |
|--------|------------|--------------|
| スコープ | このリポジトリ | 複数のリポジトリ |
| オンボーディング | リポジトリの構造 | ハブのレジストリ |
| 計画の対象 | ローカルのファイル | 子計画 |
| 状態 | ローカルの `.dwp/` | リポジトリ横断の `.dwp/` |
