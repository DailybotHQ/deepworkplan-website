# Archetypes

**Versi 1.1.** DWP mengenali tiga arketipe. Arketipe menentukan cara sebuah agent meng-onboard, cara rencana ditetapkan lingkupnya, dan apakah lapisan status yang dapat dibaca mesin diwajibkan.

> **Tambahan di v1.1.** **Ruang kerja agent** (§3) bergabung sebagai arketipe ketiga: rumah long-lived dari sebuah agent otonom. Git menjadi RECOMMENDED daripada diasumsikan. Dua arketipe v1.0 tidak berubah.

## Repositori individual

Sebuah basis kode yang mandiri — sebuah aplikasi, sebuah pustaka, atau sebuah layanan. Rencana beroperasi langsung pada kode.

Karakteristik:

- Satu basis kode yang koheren.
- Rencana memodifikasi berkas di repositori ini.
- Ruang kerja `.dwp/` di akar repositori.

## Orchestrator hub

Sebuah repositori koordinasi yang mengelola beberapa repositori turunan. Rencana dapat memunculkan rencana turunan di sub-repositori.

Karakteristik:

- Mengoordinasikan beberapa sub-repositori.
- Rencana dapat mendelegasikan ke rencana turunan.
- Memelihara sebuah registri repositori yang dikelola.
- Ruang kerja `.dwp/` di akar hub melacak status lintas repositori.

## Ruang kerja agent

Sebuah ruang kerja agent adalah rumah kerja long-lived dari sebuah agent otonom — ruang kerja OpenClaw, direktori layanan Hermes, direktori data daemon asisten pribadi, atau volume persisten cloud agent. Ini adalah ruang kerja, tidak harus sebuah repositori git, dan produk utamanya adalah pekerjaan berkelanjutan agent daripada satu basis kode.

Wawasan yang membuat arketipe ini mungkin: **harness adalah sebuah ruang kerja, bukan khususnya sebuah repositori.** Setiap elemen harness yang didefinisikan metodologi untuk repositori memiliki padanan ruang kerja langsung:

| Elemen harness (repositori) | Padanan ruang kerja agent |
|------------------------------|---------------------------|
| `AGENTS.md` (aturan, perintah cepat) | Konteks berdiri ruang kerja — `AGENTS.md` itu sendiri, atau berkas standing-orders platform |
| `docs/` (pengetahuan tahan lama) | Berkas pengetahuan ruang kerja dan dokumen memori |
| `.agents/` (skill, agent, command) | Direktori skill platform — OpenClaw secara native memindai `<workspace>/.agents/skills/` |
| `.dwp/` (rencana, draf) | `.dwp/` di akar ruang kerja — tidak berubah |
| log git (status, dapat dilanjutkan) | `state.json` per rencana ([Status rencana](/spec/plan-state)), REQUIRED di sini |

Sebuah ruang kerja agent MUST menyediakan `AGENTS.md`, `.agents/`, dan `.dwp/` di akar ruang kerja.

**Git RECOMMENDED, tidak REQUIRED.** Di mana git tidak ada, setiap rencana MUST membawa lapisan status yang dapat dibaca mesin: checkpoint `state.json`, catatan gate, dan timestamp per-tugas membawa informasi pemulihan yang dibawa log git di sebuah repositori.

Rencana di ruang kerja agent biasanya berjalan tanpa pengawasan: giliran heartbeat atau cron terjadwal melanjutkan rencana terbuka melalui Protokol Resume DWP, mengeksekusi tugas atomik berikutnya, memperbarui lapisan status, dan yield. Rencana — bukan sesi — adalah unit kontinuitas.

## Heuristik klasifikasi

<ArchetypesTree lang="en" class="not-prose my-10" />

Klasifikasikan sebagai **ruang kerja agent** terlebih dahulu, ketika target adalah direktori kerja platform agent otonom — sinyal: berkas identitas platform (seperti `SOUL.md` atau `HEARTBEAT.md` milik OpenClaw), tidak ada stack aplikasi utama, dan konten yang didominasi status milik agent itu sendiri. Satu sinyal platform yang kuat sudah cukup.

Jika tidak, klasifikasikan sebagai **orchestrator hub** ketika mayoritas jelas dari hal berikut berlaku: beberapa repositori git bersarang atau submodule; sebuah registri atau manifes repositori yang dikelola; konfigurasi yang menunjuk ke repositori eksternal. Klasifikasikan sebagai **repositori individual** jika tidak — itu adalah nilai default yang aman.

Ketika sinyal ambigu, agent MUST menyajikan penilaian dan buktinya kepada pengguna dan meminta konfirmasi sebelum melanjutkan.

## Perbedaan onboarding

| Aspek | Individual | Orchestrator | Ruang kerja agent |
|--------|------------|--------------|-----------------|
| Lingkup | Repositori ini | Beberapa repositori | Ruang kerja dan rencananya |
| Onboarding | Struktur repositori | Registri hub | Berkas platform + konvensi ruang kerja |
| Target rencana | Berkas lokal | Rencana turunan | Repositori lokal atau eksternal |
| Status | `.dwp/` lokal | `.dwp/` lintas repositori | `.dwp/` + `state.json` (REQUIRED tanpa git) |
| Git | Required | Required | RECOMMENDED |
| Lapisan status | RECOMMENDED | RECOMMENDED | REQUIRED tanpa git; REQUIRED untuk jalannya tanpa pengawasan |
