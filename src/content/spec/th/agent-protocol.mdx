# โปรโตคอลของเอเจนต์

**เวอร์ชัน 1.0** โปรโตคอลนี้นิยามว่าเอเจนต์เขียนโค้ด AI MUST ประพฤติอย่างไรเมื่อทำงานกับ Deep Work Plan คำสำคัญ MUST, SHOULD และ MAY เป็นไปตาม RFC 2119

<AgentLifecycle lang="th" class="not-prose my-10" />

## การออนบอร์ด

ก่อนสร้างหรือดำเนินแผน เอเจนต์ MUST ออนบอร์ดเข้ากับ repository การออนบอร์ดอิงการให้เหตุผล ไม่ใช่อิงสคริปต์ เอเจนต์อ่านโครงสร้าง เอกสาร และการกำหนดค่าของ repository เพื่อสร้างแบบจำลองความคิด

เอเจนต์ SHOULD ระบุ

- archetype ของ repository (แบบเดี่ยวเทียบกับแบบ orchestrator)
- คำสั่ง build, test และ lint
- ข้อตกลงที่มีอยู่สำหรับสไตล์ โครงสร้าง และการตั้งชื่อ
- skill และ agent ที่มีอยู่

## การวางแผน

เมื่อสร้างแผน เอเจนต์ MUST

- แยกย่อยเป้าหมายเป็นงานตามลำดับที่ทบทวนได้
- เขียนแต่ละงานด้วยโครงสร้างเก้าส่วน
- จบด้วยสองงานสุดท้ายที่บังคับ
- ถามคำถามเพื่อให้ชัดเจนเมื่อเป้าหมายกำกวม

## การดำเนินงาน

ระหว่างการดำเนินงาน เอเจนต์ MUST

- อ่านแผนทั้งหมดก่อนเริ่ม
- ดำเนินงานตามลำดับ เว้นแต่ส่วนพึ่งพิงอนุญาตเป็นอื่น
- อัปเดต `PROGRESS.md` หลังแต่ละงาน
- ทำเครื่องหมายสถานะงานอย่างแม่นยำ
- หยุดและถามเมื่อติดขัดแทนที่จะเดา

## การปรับแต่ง

เมื่อปรับแต่ง เอเจนต์ MUST รักษางานที่เสร็จแล้วไว้ อัปเดตตารางงาน และบันทึกสิ่งที่เปลี่ยนแปลง

## การดำเนินต่อ

เมื่อดำเนินต่อ เอเจนต์ MUST สร้างสถานะขึ้นใหม่จาก `PROGRESS.md` และไฟล์งานก่อนดำเนินการต่อ

## การสื่อสาร

เอเจนต์ SHOULD รายงานอย่างกระชับ รายงานสถานะ MUST แยกแยะงานที่เสร็จแล้ว งานที่กำลังทำ และงานที่ค้างอยู่

## ความปลอดภัย

เอเจนต์ MUST NOT commit ความลับ MUST คง `.dwp/` ไว้ใน gitignore และ SHOULD ถามก่อนทำปฏิบัติการที่ทำลายล้าง
