# มาตรฐานเอกสาร

**เวอร์ชัน 1.0** มาตรฐานนี้นิยามว่า Deep Work Plan จัดทำเอกสารโครงสร้าง งาน และความคืบหน้าอย่างไร มันใช้กับทุกแผนที่สร้างขึ้นภายใต้ระเบียบวิธี DWP คำสำคัญ MUST, SHOULD และ MAY ใช้ตามที่นิยามไว้ใน RFC 2119

## README ของแผน

ทุกแผน MUST มี `README.md` ที่ประกอบด้วย

- **ชื่อเรื่อง** — `# Deep Work Plan: <name>`
- **เป้าหมาย** — คำบอกกล่าวแบบร้อยแก้วถึงวัตถุประสงค์ของแผน
- **วัสดุต้นทาง** — ลิงก์หรือพาธไปยังอินพุตมาตรฐาน (เป็นทางเลือก)
- **งาน** — ตาราง markdown ที่มีหมายเลขงาน ชื่อ และช่องทำเครื่องหมายสถานะ
- **สถานะ** — บรรทัดในรูปแบบ `<n>/<total> tasks complete`

## ไฟล์งาน

ไฟล์งานแต่ละไฟล์ MUST ตั้งชื่อว่า `<n>.task_<slug>.md` และมีโครงสร้างเก้าส่วน

## PROGRESS.md

`PROGRESS.md` คือ log การทำงานแบบเพิ่มได้อย่างเดียว แต่ละรายการ MUST บันทึก

- timestamp แบบ ISO 8601
- หมายเลขและชื่องาน
- สิ่งที่ทำไป
- การเบี่ยงเบนหรือเหตุผลในการข้ามใด ๆ

## เครื่องหมายสถานะ

- `[ ]` — ยังไม่เริ่ม
- `[~]` — กำลังทำ
- `[x]` — เสร็จ
- `[!]` — ติดขัด

## หัวข้อ

หัวข้อทั้งหมด MUST ใช้ตัวพิมพ์แบบประโยค เอกสาร SHOULD หลีกเลี่ยงภาษาเชิงการตลาดและเครื่องหมายอัศเจรีย์

## สองงานสุดท้ายที่บังคับ

ทุกแผน MUST จบด้วยสองงานมาตรฐาน

1. **การค้นพบ Skills และ Agents** — ระบุ skill และ agent ที่นำกลับมาใช้ใหม่ได้ซึ่งถูกสร้างขึ้น
2. **รายงานสรุปสำหรับผู้บริหาร** — บทสรุปผลลัพธ์สำหรับผู้มีส่วนได้ส่วนเสีย
