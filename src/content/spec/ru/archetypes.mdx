# Архетипы

**Версия 1.1.** DWP распознаёт три архетипа. Архетип определяет, как агент проходит онбординг, как очерчиваются границы планов и является ли машиночитаемый слой состояния обязательным.

> **Аддитивно в v1.1.** **Рабочее пространство агента** (§3) присоединяется как третий архетип: долгоживущий дом автономного агента. Git становится RECOMMENDED, а не обязательным. Два архетипа v1.0 остаются без изменений.

## Отдельный репозиторий

Самодостаточная кодовая база — приложение, библиотека или сервис. Планы работают непосредственно над кодом.

Характеристики:

- Единая целостная кодовая база.
- Планы изменяют файлы в этом репозитории.
- Рабочее пространство `.dwp/` в корне репозитория.

## Хаб-оркестратор

Координирующий репозиторий, управляющий несколькими дочерними репозиториями. Планы могут порождать дочерние планы в суб-репозиториях.

Характеристики:

- Координирует несколько суб-репозиториев.
- Планы могут делегировать дочерним планам.
- Ведёт реестр управляемых репозиториев.
- Рабочее пространство `.dwp/` в корне хаба отслеживает межрепозиторное состояние.

## Рабочее пространство агента

Рабочее пространство агента — это долгоживущий рабочий дом автономного агента: рабочее пространство OpenClaw, служебная директория Hermes, директория данных демона личного помощника или постоянный том облачного агента. Это рабочее пространство, не обязательно git-репозиторий, и его основной продукт — текущая работа агента, а не одна кодовая база.

Понимание, делающее этот архетип возможным: **harness — это рабочее пространство, не обязательно репозиторий.** Каждый элемент harness, который методология определяет для репозитория, имеет прямой эквивалент в рабочем пространстве:

| Элемент harness (репозиторий) | Эквивалент в рабочем пространстве агента |
|-------------------------------|------------------------------------------|
| `AGENTS.md` (правила, быстрые команды) | Постоянный контекст рабочего пространства — сам `AGENTS.md` или файл постоянных инструкций платформы |
| `docs/` (долговечные знания) | Файлы знаний рабочего пространства и документы памяти |
| `.agents/` (навыки, агенты, команды) | Директория навыков платформы — OpenClaw нативно сканирует `<workspace>/.agents/skills/` |
| `.dwp/` (планы, черновики) | `.dwp/` в корне рабочего пространства — без изменений |
| git log (состояние, возобновляемость) | `state.json` для каждого плана ([Состояние плана](/spec/plan-state)), REQUIRED здесь |

Рабочее пространство агента MUST предоставлять `AGENTS.md`, `.agents/` и `.dwp/` в корне рабочего пространства.

**Git RECOMMENDED, но не REQUIRED.** Там, где git отсутствует, каждый план MUST нести машиночитаемый слой состояния: контрольная точка в `state.json`, gate-записи и пер-задачные метки времени несут информацию для восстановления, которую git log несёт в репозитории.

Планы в рабочем пространстве агента, как правило, выполняются автономно: запланированное heartbeat или cron-поворот возобновляет открытый план через Протокол возобновления DWP, выполняет следующую атомарную задачу, обновляет слой состояния и завершается. Единицей непрерывности является план, а не сессия.

## Эвристика классификации

<ArchetypesTree lang="ru" class="not-prose my-10" />

Сначала классифицируйте как **рабочее пространство агента**, если цель — рабочая директория автономной агентной платформы: сигналы — файл идентичности платформы (например, `SOUL.md` или `HEARTBEAT.md` OpenClaw), отсутствие основного стека приложения и контент, являющийся преимущественно собственным состоянием агента. Достаточно одного сильного сигнала платформы.

В противном случае классифицируйте как **хаб-оркестратор**, когда явное большинство следующих признаков присутствует: несколько вложенных git-репозиториев или субмодулей; реестр или манифест управляемых репозиториев; конфигурация, указывающая на внешние репозитории. В остальных случаях классифицируйте как **отдельный репозиторий** — это безопасное умолчание.

Когда сигналы неоднозначны, агент MUST представить свою оценку и свидетельства пользователю и запросить подтверждение перед продолжением.

## Различия в онбординге

| Аспект | Отдельный | Оркестратор | Рабочее пространство агента |
|--------|-----------|-------------|------------------------------|
| Границы | Этот репозиторий | Несколько репозиториев | Рабочее пространство и его планы |
| Онбординг | Структура репозитория | Реестр хаба | Файлы платформы + соглашения рабочего пространства |
| Цель плана | Локальные файлы | Дочерние планы | Локальные или внешние репозитории |
| Состояние | Локальный `.dwp/` | Межрепозиторный `.dwp/` | `.dwp/` + `state.json` (REQUIRED без git) |
| Git | Обязателен | Обязателен | RECOMMENDED |
| Слой состояния | RECOMMENDED | RECOMMENDED | REQUIRED без git; REQUIRED для автономных запусков |
