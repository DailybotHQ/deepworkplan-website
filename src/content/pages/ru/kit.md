## Набор Deep Work Plan

Набор — это всё необходимое, чтобы применять методологию на практике. Он устанавливается из
`DailybotHQ/deepworkplan-skill`:

```bash
npx skills add DailybotHQ/deepworkplan-skill
```

### Навык и его под-навыки

Навык Deep Work Plan — это маршрутизатор плюс восемь под-навыков:

- **create** — разложить цель в структурированный план (`/dwp-create`).
- **execute** — выполнять план задача за задачей, проверяя каждый gate (`/dwp-execute`).
- **refine** — добавлять, удалять или переупорядочивать задачи, сохраняя завершённую работу (`/dwp-refine`).
- **resume** — восстановить состояние и продолжить прерванный план (`/dwp-resume`).
- **status** — отчитаться о прогрессе, ничего не меняя (`/dwp-status`).
- **verify** — объективно проверить соответствие репозитория и плана (`/dwp-verify`).
- **onboard** — сделать репозиторий AI-first (`/deepworkplan-onboard`).
- **author** — создать или развить собственные навыки, агентов и команды репозитория (`/skill-create`, `/agent-create`).

### Команды

Тонкие слеш-команды делегируют под-навыкам и дополнениям:

- `dwp-create`, `dwp-execute`, `dwp-refine`, `dwp-resume`, `dwp-status`, `dwp-verify` — цикл «планирование — выполнение — проверка».
- `skill-create`, `agent-create` — делегируют под-навыку author.
- `lib-upgrade` — делегирует дополнению dependency-upgrade (устанавливается, только когда это дополнение принято).

### Адаптеры

Тонкие интеграции для отдельных агентов: Claude Code, Cursor, OpenAI Codex, GitHub Copilot, Google Gemini, OpenCode, Windsurf, Cline, Antigravity, OpenClaw, Hermes и облачных/фоновых агентов (удалённые задачи Claude Code, Codex cloud, агенты класса Jules). OpenClaw и Hermes — автономные агентные платформы, выполняющие планы под профилем автономного выполнения по расписанию heartbeat или cron.

### Пресеты онбординга

Руководства-рассуждения по отдельным стекам, которые процесс onboard использует для адаптации документации, навыков и команд валидации, —
никогда не шаблоны. Шесть пресетов: Django, Vue + Vite, Astro/Svelte, Node/TS service, Python package/CLI
и запасной generic.

### Дополнения (по желанию)

Опциональные возможности, которые процесс onboard может добавить в репозиторий, — никогда не входящие в базовый набор AI-first:

- **Devcontainer** — воспроизводимый, изолированный dev-контейнер с устойчивой аутентификацией AI-CLI.
- **Dailybot** — отчётность о прогрессе и вехах по принципу best-effort для команд, использующих Dailybot.
- **Dependency upgrade** — независимые от менеджера пакетов, партийные, проверяемые, обратимые обновления.
- **Design system** — ограниченный поверхностями интерфейса `DESIGN.md` (по пути `docs/DESIGN.md`, на который ссылается `AGENTS.md`), выведенный из реального источника дизайна репозитория, с профилями для визуального UI, стилизованного вывода CLI и диалогового обмена сообщениями, чтобы агенты генерировали фирменный интерфейсный вывод; визуальный профиль включён по умолчанию при обнаружении дизайн-системы, профили CLI и диалоговый рекомендуются при обнаружении, и о них всегда спрашивают.
- **AI Diff Reviewer** — усиливает Security Review структурированным локальным обзором ([AI Diff Reviewer](https://github.com/DailybotHQ/ai-diff-reviewer) v2 + обязательный `.review/extension.md`); опциональный Flow B добавляет CI-гейт слияния PR, использующий ту же extension. Всегда спрашивайте Flow A или Flow B; никогда не устанавливайте автоматически.
