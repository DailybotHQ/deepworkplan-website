## Швидкий старт

Встановіть скіл, проведіть онбординг репозиторію, потім плануйте та виконуйте з будь-яким агентом — кроки, що роблять
ваш репозиторій spec-driven та пілотовним агентами.

## Шлях прийняття

### 1. Встановіть скіл Deep Work Plan

Додайте скіл до вашого репозиторію. Він постачає маршрутизатор плюс вісім суб-скілів — create, execute, refine,
resume, status, verify, onboard та author. Skills CLI — найшвидший шлях:

```bash
npx skills add DailybotHQ/deepworkplan-skill
```

Або склонуйте репозиторій та запустіть setup там, де доступні git і shell:

```bash
git clone https://github.com/DailybotHQ/deepworkplan-skill.git && cd deepworkplan-skill && ./setup.sh
```

### 2. Проведіть онбординг репозиторію

Запустіть суб-скіл onboard і дайте агенту поміркувати про ваш реальний репозиторій — його стек, менеджер
пакетів та реальні валідаційні команди:

```bash
/deepworkplan-onboard
```

Він генерує `AGENTS.md`, базу знань `docs/`, документацію кожного модуля та крос-агентний дім `.agents/`
(із символьним посиланням `.claude → .agents`), підключає тонкі команди `dwp-*` та створює `.dwp/` у gitignore
для планів і чернеток. Нічого не шаблонізується; усе адаптується до вашого репозиторію.

### 3. Розвивайте набір та приймайте add-on

Використовуйте `/skill-create` та `/agent-create` (суб-скіл author), щоб вирощувати доречні для стека скіли,
агенти та команди. Онбординг також пропонує чотири add-on із явним прийняттям — devcontainer, Dailybot,
dependency-upgrade та design-system, — які ви приймаєте лише тоді, коли вони пасують. Репозиторій повністю відповідний з нульовою кількістю add-on.

### 4. Плануйте та виконуйте

Згенеруйте Deep Work Plan і виконуйте його завдання за завданням:

```bash
/dwp-create <goal>
/dwp-execute
```

Використовуйте `/dwp-status`, `/dwp-refine` та `/dwp-resume`, поки робота просувається. Кожен план несе пронумеровані завдання,
валідаційні gate та протокол завершення, тож робота лишається структурованою та відновлюваною між сесіями.

## Результат

Ваш репозиторій стає spec-driven та пілотовним агентами: план є стійким джерелом істини, а
сам репозиторій стає harness, за яким виконує будь-який агент.
