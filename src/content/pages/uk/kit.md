## Набір Deep Work Plan

Набір — це все, що потрібно, щоб запустити методологію на практиці. Він встановлюється з
`DailybotHQ/deepworkplan-skill`:

```bash
npx skills add DailybotHQ/deepworkplan-skill
```

### Скіл та його суб-скіли

Скіл Deep Work Plan — це маршрутизатор плюс вісім суб-скілів:

- **create** — розкласти мету на структурований план (`/dwp-create`).
- **execute** — виконати план завдання за завданням, перевіряючи кожен gate (`/dwp-execute`).
- **refine** — додати, видалити чи переставити завдання, зберігаючи завершену роботу (`/dwp-refine`).
- **resume** — відтворити стан та продовжити перерваний план (`/dwp-resume`).
- **status** — відзвітувати про прогрес без жодних змін (`/dwp-status`).
- **verify** — обʼєктивно перевірити відповідність репозиторію та плану (`/dwp-verify`).
- **onboard** — зробити репозиторій AI-first (`/deepworkplan-onboard`).
- **author** — створити чи розвинути власні скіли, агенти та команди репозиторію (`/skill-create`, `/agent-create`).

### Команди

Тонкі слеш-команди делегують суб-скілам та add-on:

- `dwp-create`, `dwp-execute`, `dwp-refine`, `dwp-resume`, `dwp-status`, `dwp-verify` — цикл план-виконання-перевірка.
- `skill-create`, `agent-create` — делегують суб-скілу author.
- `lib-upgrade` — делегує add-on dependency-upgrade (встановлюється лише тоді, коли цей add-on прийнято).

### Адаптери

Тонкі інтеграції під кожного агента для Claude Code, Cursor, OpenAI Codex, GitHub Copilot, Google Gemini, OpenCode, Windsurf, Cline, Antigravity, OpenClaw, Hermes та хмарних і фонових агентів (віддалені завдання Claude Code, Codex cloud, агенти класу Jules). OpenClaw та Hermes — автономні агентні платформи, що виконують плани під профілем автономного виконання, керованого heartbeat або cron-плануванням.

### Пресети онбордингу

Орієнтири міркування під кожен стек, які потік onboard використовує для адаптації документації, скілів та валідаційних команд —
ніколи не шаблони. Шість пресетів: Django, Vue + Vite, Astro/Svelte, Node/TS-сервіс, Python-пакет/CLI
та загальний запасний варіант.

### Add-on (явне прийняття)

Опціональні можливості, які потік onboard може нашарувати на репозиторій, — ніколи не частина AI-first базису:

- **Devcontainer** — відтворюваний, ізольований dev-контейнер зі стійкою автентифікацією AI-CLI.
- **Dailybot** — найкраще-можливе звітування про прогрес та віхи для команд, що використовують Dailybot.
- **Dependency upgrade** — незалежні від менеджера пакетів, партіями, перевірені, відкочувані оновлення.
- **Дизайн-система** — обмежений фронтендом `DESIGN.md` (за шляхом `docs/DESIGN.md`, з посиланням з `AGENTS.md`), осмислений із реальних дизайн-токенів репозиторію, щоб агенти генерували UI у стилі бренду; типово ввімкнений, коли виявлено дизайн-систему.
