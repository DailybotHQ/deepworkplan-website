## มันคืออะไร

Deep Work Plan (DWP) คือระเบียบวิธี ไม่ใช่ผลิตภัณฑ์ มันนิยามวิธีเปลี่ยนเป้าหมายให้เป็นแผนที่ตกลงร่วมกัน แยกแผนนั้นเป็นงานแบบ atomic ที่ตรวจสอบได้อย่างอิสระ และรันแต่ละงานในลูปที่จดจ่อซึ่งจบด้วยการตรวจสอบ

มันเป็นระเบียบวิธีที่ขับเคลื่อนด้วยข้อกำหนด แผนคือแหล่งความจริงที่ยั่งยืน และเอเจนต์ดำเนินการเทียบกับเกณฑ์การยอมรับและ validation gate ที่ชัดเจน ต่างจากระบบขับเคลื่อนด้วยข้อกำหนดที่ผูกติดกับเครื่องมืออย่าง GitHub Spec Kit, Amazon Kiro หรือ Tessl DWP เป็นกลางต่อเครื่องมือและฝังอยู่ใน repository โดยกำเนิด มันยังเป็น harness engineering ที่ทำให้พกพาได้ — บริบท guardrail และสถานะที่ดำเนินต่อได้ซึ่งทำให้เอเจนต์เชื่อถือได้ถูกติดตั้งลงใน repository เอง (AGENTS.md, docs, บ้าน skill .agents/, skill ของ DWP) เอเจนต์ใดก็ขับเคลื่อน repo ใดก็ได้

มันจงใจเป็นกลางว่าคุณใช้เอเจนต์ AI หรือเทคสแตกใด — adapter แปลลูปหลักเดียวกันไปยัง Claude, Cursor, Copilot, Codex, Gemini และอื่น ๆ แผน งาน และ log การทำงานล้วนเป็น Markdown ธรรมดา งานจึงยังคงอ่านได้ ทบทวนได้ และควบคุมเวอร์ชันได้

## หลักการสำคัญ

- **วางแผนก่อนดำเนินการ** — ไม่มีการเขียนโค้ดจนกว่าแผนจะตกลงร่วมกัน แผนคือสัญญาระหว่างคุณกับเอเจนต์ [อ่านระเบียบวิธี](/methodology)
- **งานเป็นแบบ atomic** — แต่ละงานถูกกำหนดขอบเขตให้ดำเนินการและตรวจสอบได้ด้วยตัวเอง แล้ว commit แบบ atomic [ดูข้อกำหนด](/spec)
- **ตรวจสอบทุกอย่าง** — ทุกงานจบด้วยการตรวจสอบที่ชัดเจนก่อนงานถัดไปจะเริ่ม โดยบันทึกความคืบหน้าไว้ใน git [สำรวจ kit](/kit)

## โดยสรุป

- ระเบียบวิธีแบบเปิด ใช้ใบอนุญาต MIT
- เป็นกลางต่อเฟรมเวิร์กและเอเจนต์
- ดูแลโดย Dailybot และชุมชน
- รวมข้อกำหนด คำสั่ง adapter preset และตัวอย่าง
- markdown ล้วน — ไม่มี runtime ไม่มีการล็อกอิน
- เปลี่ยนทุก repository ให้เป็นโค้ดเบสแบบ AI-first ที่เอเจนต์ขับเคลื่อนได้
