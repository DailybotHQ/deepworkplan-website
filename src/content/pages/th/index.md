## โมเดลสำคัญ บริบทสำคัญยิ่งกว่า

Deep Work Plan เปลี่ยนทุก repository ให้เป็นสภาพแวดล้อมที่มีโครงสร้าง — บริบท guardrail และแผนที่ยั่งยืน — ซึ่งเอเจนต์เขียนโค้ดใดก็ดำเนินการได้อย่างแม่นยำและทำงานระยะยาวให้เสร็จได้

Deep Work Plan (DWP) คือระเบียบวิธีและ kit แบบเปิดที่ใช้ใบอนุญาต MIT สำหรับการวางแผนและดำเนินงานซอฟต์แวร์ที่ซับซ้อนด้วยเอเจนต์ AI คุณไม่ต้องเลือกวิธีติดตั้งหรือคัดลอกเทมเพลต — คุณคัดลอก prompt ของ init.md และวางลงใน agent ของคุณ:

> คัดลอก prompt ของ init.md และวางลงใน coding agent ของคุณ — Claude Code, Cursor, Codex หรืออื่น ๆ — เพื่อทำให้ repository ใดก็ได้เป็น AI-first

> Deep Work Plan คือ spec-driven development ที่ repository เองกลายเป็น harness

- [อ่านระเบียบวิธี](/methodology)

## ปัญหาและคำตอบ

เอเจนต์เขียนโค้ด AI มีประสิทธิภาพอย่างน่าทึ่งในช่วงสั้น ๆ แต่กับงานระยะยาว — การย้ายระบบ ระบบย่อยใหม่ การรีแฟกเตอร์ข้ามไฟล์หลายสิบไฟล์ — มันเบี่ยงเบน บริบทเต็ม การตัดสินใจถูกลืม และงานหลายชั่วโมงถูกทิ้งกลางคัน

Deep Work Plan ตอบด้วย **spec-driven development** แผนคือแหล่งความจริงที่ยั่งยืน และเอเจนต์ดำเนินการเทียบกับเกณฑ์การยอมรับและ validation gate ที่ชัดเจน การเบี่ยงเบนลดลง งานยังคงตรวจสอบได้ และเอเจนต์ใดก็ดำเนินต่อได้ข้ามเซสชัน ต่างจากเครื่องมือขับเคลื่อนด้วยข้อกำหนดที่ผูกติดกับ IDE หรือผู้ขายตัวใดตัวหนึ่ง (GitHub Spec Kit, Amazon Kiro, Tessl) DWP เป็นกลางต่อเครื่องมือและฝังอยู่ใน repository โดยกำเนิด

มันยังเป็น **harness engineering** ที่ทำให้พกพาได้ agent harness คือโครงนั่งร้านรอบโมเดล — บริบท เครื่องมือ control loop guardrail สถานะที่ดำเนินต่อได้ — ที่ทำให้มันเชื่อถือได้ Deep Work Plan ติดตั้ง harness นั้นลงใน repository เอง เอเจนต์ใดก็ขับเคลื่อน repo ใดก็ได้

## สิ่งที่คุณได้รับ

- AGENTS.md ที่รากของ repository ผ่านการให้เหตุผลจากเทคสแตกและคำสั่งจริงของคุณ
- `docs/` ที่จัดหมวดหมู่และเอกสารต่อโมดูล
- ไดเรกทอรี `.agents/` (skill, agent, คำสั่ง) พร้อม symlink จาก `.claude` ไปยัง `.agents`
- skill ของ Deep Work Plan ติดตั้งครั้งเดียวสำหรับทุกเอเจนต์ — รวมถึง sub-skill ชื่อ author เพื่อให้ repository ปลูก skill, agent และคำสั่งของตัวเองได้ พร้อมส่วนเสริมแบบเลือกเข้าร่วมอย่าง dependency-upgrade และส่วนเสริม design-system ฝั่ง frontend
- แผนระยะยาวที่ดำเนินต่อได้ในโฟลเดอร์ `.dwp/` ที่ถูก gitignore

## เกิดอะไรขึ้นเมื่อคุณรันมัน

1. **เอเจนต์ของคุณเปิด [/init.md](/init.md)** — มันอ่านพรอมต์การออนบอร์ดและระเบียบวิธี ข้อกำหนด และ kit ที่ลิงก์ไปถึง
2. **มันติดตั้ง skill ของ Deep Work Plan** — เครื่องยนต์ที่เหมือนกันในทุก repository ได้แก่ ตัวกำหนดเส้นทางพร้อม sub-skill (create, execute, refine, resume, status, verify, onboard, author) สำหรับ Claude Code, Cursor, Codex, Gemini และ Copilot
3. **มันปรับ repository ของคุณ** — โดยให้เหตุผลเกี่ยวกับเทคสแตกจริงของคุณ (ไม่เคยคัดลอกวาง) มันเขียน AGENTS.md, `docs/`, README ต่อโมดูล, kit `.agents/` ที่ผ่านการให้เหตุผล และ `.dwp/` ที่ถูก gitignore repository ของคุณกลายเป็น harness
4. **คุณวางแผนและดำเนินการ** — Deep Work Plan ระยะยาว รันทีละขั้นเทียบกับเกณฑ์การยอมรับและ validation gate ที่ชัดเจน อย่างอิสระเป็นเวลาหลายชั่วโมง

skill คือเครื่องยนต์ที่ใช้ซ้ำได้ ติดตั้งในแบบเดียวกันทุกที่ สิ่งที่ถูกปรับคือ repository ของคุณ กลไกการติดตั้ง (Skills CLI, OpenClaw หรือ git clone) อยู่ในพรอมต์ [/init.md](/init.md) และบน[quickstart](/quickstart)
