## ทำไมคุณถึงวางใจมันได้

ไม่ควรมีใครติดตั้งสกิลที่ตัวเองเชื่อถือไม่ได้ Deep Work Plan ถูกสร้างให้ตรวจสอบได้ ไม่ใช่ให้เชื่อโดยสุจริต: โอเพนซอร์ส ใช้ Markdown เป็นหลัก ไม่ทำลายข้อมูล และตรวจสอบได้ก่อนรัน

- **โอเพนซอร์สและอนุญาตสิทธิ์ MIT** — ทั้งเว็บไซต์และสกิลเป็นสาธารณะและสามารถ diff ได้ อ่านทุกบรรทัดก่อนรัน
- **Markdown เป็นหลัก — ไม่มีเครือข่าย ไม่มีการเก็บข้อมูลการใช้งาน** — ไม่มี CLI ไม่มี HTTP API ไม่มีขั้นตอนการยืนยันตัวตน ไม่เรียกเครือข่ายและไม่ส่งข้อมูลการใช้งาน ไม่มีข้อมูลใด ๆ เกี่ยวกับ repository ของคุณออกจากเครื่องของคุณ
- **ไม่ทำลายข้อมูลตามการออกแบบ** — สิ่งเดียวที่สกิลทำซึ่งเกี่ยวข้องกับความปลอดภัยคือการเปลี่ยน repository ของคุณ และมันประสานแทนที่จะเขียนทับ ตรวจสอบว่ามีอะไรอยู่แล้ว เสนอแผน และถามก่อนแทนที่สิ่งใด ผลลัพธ์ของแผนอยู่ในไดเรกทอรี `.dwp/` ที่ถูก gitignore ไว้
- **ไม่แตะความลับ** — ไม่เคยคอมมิตความลับ ต่อท้าย `.gitignore` แทนที่จะเขียนใหม่ และเก็บการเปลี่ยนแปลงไว้ใน diff เล็ก ๆ ที่ตรวจทานได้
- **สามารถตรวจสอบแหล่งที่มาได้** — ทุก release เผยแพร่ checksum สำหรับสกิลที่ส่งมา

## ตรวจสอบก่อนรัน

ปฏิบัติต่อสกิลว่าไม่น่าเชื่อถือจนกว่าคุณจะตรวจสอบแล้ว ทุก release แนบไฟล์ `SHA256SUMS` ที่ครอบคลุมสกิลที่ส่งมา ดาวน์โหลดสำหรับเวอร์ชันที่คุณต้องการติดตั้ง แล้วตรวจสอบว่าสำเนาของคุณตรงกัน ค่า exit ที่ไม่ใช่ศูนย์หมายความว่าไฟล์ไม่ตรงกันและคุณควรหยุด

```bash
git clone https://github.com/DailybotHQ/deepworkplan-skill.git
cd deepworkplan-skill
# Download the checksums for the release you intend to install (replace vX.Y.Z):
curl -fsSL -o SHA256SUMS \
  https://github.com/DailybotHQ/deepworkplan-skill/releases/download/vX.Y.Z/SHA256SUMS
./setup.sh --verify        # non-zero exit means a file does not match — stop
```

Release ถูก**ตรวจสอบด้วย checksum ไม่ใช่ลงนาม** — การลงนาม (cosign หรือ maintainer GPG) เป็นขั้นตอนถัดไปที่ได้รับการจัดทำเป็นเอกสาร ไม่ใช่สิ่งที่อ้างในปัจจุบัน เนื่องจากทุกอย่างเปิดเผย คุณยังสามารถ diff ไฟล์ใดก็ได้กับ repository ที่ tag ของมัน

## รายงานช่องโหว่

รายงานปัญหาด้านความปลอดภัยเป็นการส่วนตัวที่ **security@dailybot.com** แทนที่จะเปิด issue สาธารณะ เราตอบรับรายงานภายใน 3 วันทำการและมุ่งส่งแพตช์หรือมาตรการบรรเทาภายใน 30 วันสำหรับการค้นพบที่ถูกต้อง ขึ้นอยู่กับความรุนแรง เว็บไซต์และสกิลใช้นโยบายเดียวกัน

## ข้อจำกัดที่ซื่อสัตย์

- Release ถูกตรวจสอบด้วย checksum แต่ยังไม่ได้ลงนามทางเข้ารหัส การลงนามมีแผนแต่ยังไม่ได้ดำเนินการ
- Deep Work Plan รัน coding agent อิสระกับ repository ของคุณ ตรวจทานแผนที่เสนอและ diff ของมัน ระเบียบวิธีถูกออกแบบมาสำหรับการตรวจทานนั้น ไม่ใช่เพื่อแทนที่
- การอ้างสิทธิ์ความน่าเชื่อถือที่นี่อธิบายเฉพาะแหล่งทางการเท่านั้น สำเนาที่ถูกดัดแปลงหรือสำเนาจากบุคคลที่สามซึ่งแตกต่างจาก repository ไม่มีการรับประกันเหล่านี้ ตรวจสอบก่อน
